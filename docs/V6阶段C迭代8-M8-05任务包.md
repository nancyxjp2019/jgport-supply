## 2. 迭代8第五批范围（M8-05 小程序本地后端联调）
- 目标：在不引入正式微信登录的前提下，打通“开发者工具选择角色 -> 后端签发本地联调令牌 -> 小程序读取身份 -> 拉取轻量报表”的本地联调链路。
- 模块边界：本轮处理 `backend + miniprogram` 的本地联调令牌、小程序双模式切换、身份读取与轻量报表实联调；不进入真实微信登录、真机 HTTPS 联调、订单/待办/执行页面。
- 首批实现边界：
  - 后端新增 `POST /api/v1/mini-auth/dev-login`，仅在 `dev/test` 环境签发本地联调令牌。
  - 受保护接口支持 `Authorization: Bearer <token>` 的开发者工具联调方式。
  - 小程序新增 `演示模式 / 本地联调` 双模式切换。
  - 小程序本地联调模式下读取 `/access/me` 和 `/reports/light/overview`。
  - 保持客户、供应商、仓库仅能看到中文阻断提示，不暴露经营金额汇总。
- 本轮不实现：
  - 微信 `wx.login`
  - `code2session`
  - 真机预览与 HTTPS 合法域名联调
  - 生产可用的正式小程序认证体系

## 2.2 M8-05 实施约束
- 本地联调令牌仅允许在 `dev/test` 环境使用，不能替代正式认证体系。
- 不允许把服务端透传密钥直接写入小程序前端代码。
- 所有用户可见提示必须中文显示。
- 小程序本地联调仍需遵守角色边界：运营/财务/管理员可见轻量报表，客户/供应商/仓库仅展示中文阻断提示。
- 本轮只保证微信开发者工具本地联调，不承诺真机、体验版、正式版可直接复用。
