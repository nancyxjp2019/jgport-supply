## 2. 迭代8第二十批范围（M8-20 退款核销与资金驳回首批）
- 目标：落地管理后台退款核销与资金驳回台首批页面，补齐退款待审核流转、退款驳回与单据核销能力。
- 模块边界：本轮处理 `admin-web + backend资金退款与核销接口补齐 + M4既有资金确认链路复用 + docs`；不新增批量退款、不新增批量核销、不新增自动补偿重放。
- 首批实现边界：
  - 管理后台新增 `/funds-reconcile` 页面与导航入口。
  - 后端补齐退款核销最小能力：退款申请、退款审核通过、退款驳回、单据核销。
  - 页面支持按单据类型、单据状态、退款状态筛选，并回看退款金额与退款状态流转。
  - 页面支持单笔动作：发起退款审核、退款审核通过、退款驳回、执行核销。
  - 演示模式补齐退款核销 mock 数据与单测。
- 本轮不实现：
  - 批量退款、批量核销、批量驳回
  - 自动补偿重放、跨单据联动回滚
  - 退款核销趋势报表与导出

## 2.2 M8-20 实施约束
- 页面文案统一中文，沿用 `M4` 资金单据主状态：`草稿`、`待补录金额`、`已确认`、`已核销`、`已终止`。
- 退款状态首批新增流转：`未退款`、`待审核`、`驳回`、`部分退款`、`已退款`。
- 退款申请仅允许 `已确认/已核销` 单据发起，退款金额不得超过单据实收实付金额。
- 核销仅允许 `已确认` 单据执行，重复核销需返回幂等提示。
- 首批仅开放单笔审核，不在本轮扩展批量操作与自动补偿任务。
