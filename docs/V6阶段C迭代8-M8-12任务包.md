## 2. 迭代8第十二批范围（M8-12 供应商采购进度首批）
- 目标：落地供应商角色首批真实业务页，支撑供应商在小程序中查看本公司采购订单进度、回看发货准备信息，并为后续附件回传与发货确认建立入口基础。
- 模块边界：本轮处理 `backend + miniprogram + docs` 的供应商采购订单列表/详情只读能力、供应商待办真实入口、消息/待办到采购进度页的导航收口与测试；不进入付款确认、不进入异常关闭、不进入批量处理。
- 首批实现边界：
  - 后端新增供应商侧采购订单列表与详情只读接口，查询范围限定为当前供应商公司。
  - 小程序新增供应商采购进度页，展示采购订单状态、油品、数量、应付金额、来源销售订单与发货准备提示。
  - 待办页中供应商角色由“纯边界提示”升级为“真实采购进度入口”。
  - 消息中心供应商角色由“纯边界提示”升级为“采购进度提醒入口”，但仍不伪造数量。
  - 为后续附件回传预留页面入口和字段位，不在本轮落真实上传。
- 本轮不实现：
  - 供应商付款确认
  - 发货确认提交动作
  - 盖章发货指令单与附件真实上传
  - 采购订单批量列表筛选与复杂搜索
  - 供应商异常关闭、驳回、改单

## 2.2 M8-12 实施约束
- 供应商首批页面只做“查看进度 + 回看准备信息 + 保留附件入口”，不新增资金确认和异常处置动作。
- 所有供应商侧接口必须按 `supplier + supplier_company + miniprogram` 做服务端权限校验，不得依赖前端隐藏入口。
- 供应商查询范围必须严格限定为当前供应商公司，不得跨公司穿透其他供应商订单。
- 若后端暂未具备附件上传能力，前端只能给出中文占位提示，不得伪造“已上传”或“已确认发货”状态。
- 页面与消息文案必须全中文，且需突出“当前仅开放首批进度查看与发货准备回看”。

## 3. 上下游边界
- 上游输入：财务审批通过后已生成的采购订单实体、采购合同与来源销售订单关系。
- 下游输出：供应商可见的采购进度视图、待办入口、消息入口、后续附件回传预留入口。
- 外部依赖：`purchase_orders` 模型、供应商身份绑定、采购订单响应模型、小程序登录会话。
- 失败回滚：若接口权限、公司范围或字段口径不满足，则回滚本轮新增接口与页面，恢复供应商边界提示。
