## 2. 迭代8第三批范围（M8-03 登录与身份透传）
- 目标：完成管理后台最小可用登录链路、后端身份回显接口与路由守卫，为后续全业务页面挂接统一登录态。
- 模块边界：本轮只处理 `backend + admin-web` 的登录与身份透传链路；不进入小程序真实微信登录、不进入合同/订单/资金/库存业务页面。
- 首批实现边界：
  - 后端新增 `GET /api/v1/access/me`，用于返回当前已认证身份及端权限。
  - 管理后台新增登录页、演示模式角色登录、代理联调模式身份读取。
  - 管理后台路由守卫落地，未登录不得直接进入 `/dashboard`、`/board`。
  - 非开发环境禁止使用默认服务端身份透传密钥。
- 本轮不实现：
  - 微信 `wx.login` 真实换取会话
  - 小程序真实身份透传
  - 刷新 Token、过期续期、单点登录
  - 按钮级细粒度权限矩阵
