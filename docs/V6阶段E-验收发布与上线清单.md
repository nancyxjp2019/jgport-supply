# V6 阶段E-验收发布与上线清单

## 1. 文档定位
- 文档状态：`生效`
- 目标：冻结阶段 E 的验收、发布、上线、回滚与观察期执行口径，作为 V6 进入生产环境前的唯一执行清单。
- 使用边界：本文不替代 `docs/V6生产可用版本治理总纲.md`、`docs/需求方案.md` 与模块任务包；本文负责把“已经开发完成的生产化范围”转成可执行的验收与上线动作。

## 2. 当前状态
- 当前阶段状态：`未启动（基线已建立）`
- 当前结论：阶段 E 文档已建立，但必须在 `G1 ~ G5` 完成、阶段 D 重启并通过后，方可进入执行。

| 追踪矩阵 | 规则、实现、测试、验收、遗留项已完成映射 | `docs/V6规则-实现-测试-验收追踪矩阵.md` |

## 4. 业务验收清单

| 验收项 | 必须确认 |
|---|---|
| 前置治理能力 | 组织、公司、用户、角色、主数据、审批模板、参数配置可用且可追溯 |
| 订单主链 | 合同、销售订单、采购订单、收付款、仓储执行、关闭、报表主链一致 |
| 身份与权限 | 管理后台仅开放正式允许角色，小程序角色、公司归属、端权限边界正确 |
| 审计与追溯 | 关键创建、审批、确认、关闭、导出、重算、扫描动作均可回查 |
| 异常分支 | 驳回、0 金额例外、补录、校验失败、关闭阻断等异常路径已验证 |
| 用户可见文案 | 中文提示一致，且不存在演示模式、联调模式误导文案 |

## 5. 测试与质量门禁

| 类别 | 必须完成 |
|---|---|
| 后端 | 全量自动化回归、关键主链回归、权限与审计回归通过 |
| 管理后台 | 单测、构建、关键页面交互回归通过 |
| 小程序 | 语法检查、Node 测试、关键链路回归通过 |
| 手工回归 | 真实业务场景、关键角色、关键异常链至少完成一轮 |
| 文档一致性 | 需求、总控、总纲、阶段 E 清单、追踪矩阵口径一致 |
| 评审门禁 | 自检与独立评审均通过，且无阻断问题、无高优先级问题、无未处理一致性问题 |

## 6. 发布冻结清单
- 冻结上线范围、数据库迁移、参数变更、初始化数据清单、版本号与发布窗口。
- 冻结非阻断问题列表；未冻结的问题不得临时插队进发布包。
- 冻结演示/联调相关能力的去向：哪些仅保留本地、哪些在测试环境隐藏、哪些在生产彻底禁用。

## 7. 发布实施清单

| 项目 | 执行要求 |
|---|---|
| 部署包 | 后端、管理后台、小程序版本号与提交哈希一致 |
| 数据库 | 迁移脚本、执行顺序、回退方案明确 |
| 初始化 | 组织、用户、主数据、模板、参数装载顺序明确 |
| 配置 | 生产环境密钥、正式域名、HTTPS、对象存储、微信配置全部生效 |
| 审计 | 发布动作、关键配置变更、初始化装载都有留痕 |
| 发布责任 | 研发、测试、业务、运维值守人明确 |

## 8. 回滚清单

| 项目 | 必须明确 |
|---|---|
| 触发条件 | 哪些阻断问题触发回滚，谁有权下达回滚决定 |
| 回滚步骤 | 应用回滚、配置回滚、数据回滚、验证顺序 |
| 数据边界 | 哪些数据可回滚、哪些只能补偿、哪些必须人工确认 |
| 验证口径 | 回滚后需确认登录、主链、权限、审计、报表是否恢复 |

## 9. 上线后观察期清单
- 观察期内必须跟踪：登录成功率、关键主链成功率、权限异常、审计缺失、报表异常、用户阻断反馈。
- 观察期内所有新增问题必须按“阻断 / 高优先级 / 中低优先级”分级并写入书面台账。
- 观察期未通过前，不得宣称版本已稳定收口。

## 10. Go / No-Go 签署要求
- 签署角色至少包括：业务负责人、产品/方案负责人、研发负责人、测试负责人、运维负责人。
- 签署内容至少包括：
  - 本次上线范围；
  - 非目标与降级项；
  - 已知遗留问题；
  - 回滚方案；
  - Go / No-Go 结论。

## 11. 阶段 E 退出门禁
- 生产环境发布完成；
- 观察期通过；
- 阻断问题清零；
- 高优先级问题已关闭或有书面承接结论；
- 文档、审计、问题回溯资料全部归档完成。
