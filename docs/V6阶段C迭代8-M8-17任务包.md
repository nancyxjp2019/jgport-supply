## 2. 迭代8第十七批范围（M8-17 财务资金处理台首批）
- 目标：落地管理后台财务资金处理台首批页面，补齐收款单/付款单列表、详情回看、手工补录、单笔确认与凭证路径回看。
- 模块边界：本轮处理 `admin-web + backend资金单据最小查询接口补齐 + M4既有补录/确认接口复用 + docs`；不新增退款核销、不新增驳回待审核独立处理台、不新增真实文件上传。
- 首批实现边界：
  - 管理后台新增 `/funds` 页面与导航入口。
  - 后端补齐资金单据最小查询能力：收款单/付款单列表、详情只读回看。
  - 复用 `POST /api/v1/payment-docs/supplement`、`POST /api/v1/receipt-docs/supplement`、`POST /api/v1/payment-docs/{id}/confirm`、`POST /api/v1/receipt-docs/{id}/confirm`。
  - 页面支持按单据类型、状态筛选，展示合同/订单关联、金额、凭证要求、免凭证原因、退款状态与凭证路径。
  - 演示模式提供可回归的资金单据 mock 数据，覆盖补录、确认与状态推进。
- 本轮不实现：
  - 退款/核销
  - 收付款驳回/待审核独立处理台
  - 批量确认、批量补录
  - 真实附件上传，仅回看与提交凭证路径

## 2.2 M8-17 实施约束
- 页面文案统一中文，明确区分“收款单”“付款单”“待补录金额”“规则11例外”“规则14免凭证确认”等口径。
- 补录付款单必须填写 `采购合同ID + 采购订单ID + 补录金额`；补录收款单必须填写 `销售合同ID + 销售订单ID + 补录金额`。
- 单笔确认仅允许处理 `草稿` 与 `待补录金额` 状态单据，不得对 `已确认`、`已核销`、`已终止` 再次开放确认入口。
- 非0金额确认必须填写至少一个凭证路径；0金额确认继续沿用 `M4` 规则11/规则14 服务端校验，不在前端重写业务判定。
- 首批凭证能力仅处理路径数组展示与提交，不在本轮引入真实上传、预签名地址或对象存储交互。
