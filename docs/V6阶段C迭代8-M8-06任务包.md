## 2. 迭代8第六批范围（M8-06 微信登录与 code2session）
- 目标：补齐真实微信登录骨架，包括小程序 `wx.login`、后端 `code2session`、微信账号绑定表与 Bearer 登录令牌发放。
- 模块边界：本轮处理 `backend + miniprogram` 的微信登录链路、绑定表、配置项与本地绑定脚本；不进入后台绑定 UI、真机 HTTPS 联调、订单/待办/执行页面。
- 首批实现边界：
  - 后端新增 `POST /api/v1/mini-auth/wechat-login`。
  - 服务端接入微信官方 `code2session` 接口，交换 `openid/session_key`。
  - 新增 `mini_program_accounts` 绑定表，按 `openid -> 角色/公司` 解析业务身份。
  - 小程序新增 `微信登录` 模式，调用 `wx.login` 获取 `code`。
  - 开发环境提供绑定脚本，支持按 `openid` 手工创建或更新绑定。
- 本轮不实现：
  - 后台绑定管理页面
  - 真机 HTTPS 联调
  - 生产级刷新令牌 / 续期 / 单点登录
  - 手机号解密、UnionID 贯通用户中心

## 2.2 M8-06 实施约束
- 微信登录成功后仍必须经过“角色 + 公司归属 + 端权限”三层校验，不能只凭 openid 放行。
- 若微信账号未绑定业务角色，不得下发业务访问令牌。
- `debug_openid` 仅允许在 `dev/test` 环境返回，禁止在生产环境暴露。
- 小程序所有用户可见文案继续保持中文显示。
- 本轮只完成骨架，不承诺没有 `AppSecret` 时可以真实跑通。
