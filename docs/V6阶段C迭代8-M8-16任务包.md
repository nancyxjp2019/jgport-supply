## 2. 迭代8第十六批范围（M8-16 运营订单处理台首批）
- 目标：落地管理后台运营订单处理台首批页面，补齐销售订单列表、详情回看、运营审批与财务审批入口。
- 模块边界：本轮处理 `admin-web + backend已有接口复用 + docs`；不新增退款核销、不新增批量审核、不新增多维报表。
- 首批实现边界：
  - 管理后台新增 `/orders` 页面与导航入口。
  - 复用 `GET /api/v1/sales-orders`、`GET /api/v1/sales-orders/{id}`、`POST /api/v1/sales-orders/{id}/ops-approve`、`POST /api/v1/sales-orders/{id}/finance-approve`。
  - 页面支持按状态筛选订单、查看审批意见、提交运营/财务审批。
  - 演示模式提供可回归的订单审批 mock 数据。
- 本轮不实现：
  - 批量审批
  - 退款/核销
  - 收付款驳回待审核独立处理台
  - 订单附件与图谱可视化

## 2.2 M8-16 实施约束
- 页面文案统一中文，保持运营/财务语义一致。
- 运营审批仅允许处理 `待运营审批`；财务审批仅允许处理 `待财务审批`。
- 首批财务审批通过需显式填写采购合同ID、实收金额、实付金额。
- 页面仅处理单笔审批，不开放批量处理。
