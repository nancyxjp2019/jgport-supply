# 隽港顺达供应链项目 需求方案

## 1. 文档信息
- 版本：V5
- 更新日期：2026-03-05
- 状态：V5 开发基线（生效规则用于当前运行；规划规则作为本期 V5 开发依据）

## 2. 规则状态说明
- 本文档中的业务规则统一标注状态。
- `生效`：当前版本已启用，并以当前代码实现为准。
- `弃用`：历史上存在但当前版本已停用，仅保留追溯价值。
- `规划中`：已确认方向但当前版本尚未落地，不能作为上线口径。

## 3. 业务规则总表（项目强制规则）
| 规则编号 | 规则名称 | 当前状态 | 规则摘要 |
| --- | --- | --- | --- |
| BR-001 | 当前版本基线 | 生效 | 后续优化与新增需求统一以当前代码实现为基线。 |
| BR-002 | 主流程链路 | 生效 | 客户下单 -> 运营审核 -> 财务审核 -> 供应商审核 -> 仓库出货 -> 订单结束。 |
| BR-003 | 业务管理员边界 | 生效 | `ADMIN` 只查看待审与执行订单终止类动作，不直接参与运营、财务、供应商、仓库审批。 |
| BR-004 | 运营审核金额规则 | 生效 | 运营审核通过不再填写销售单价，也不再生成应收金额。 |
| BR-005 | 财务审核规则 | 生效 | 销售订单财务审核通过前必须填写收款金额并上传客户收款凭证。 |
| BR-006 | 供应商审核规则 | 生效 | 供应商审核通过必须上传盖章发货指令单。 |

| BR-008 | 驳回终止规则 | 弃用 | V4 中运营、财务、供应商、仓库驳回均为“驳回即终止”的旧规则已停用，V5 终止类动作以 BR-038、BR-042 为准。 |
| BR-009 | 客户撤销能力 | 弃用 | 客户撤销订单的旧规则已停用，不再保留前台入口。 |
| BR-010 | 供应商与仓库单据口径 | 生效 | 仓库查看供应商上传的盖章发货指令单，不再以财务节点原始提货指令单作为执行附件。 |
| BR-011 | 库存预占与回滚 | 弃用 | 旧版“财务审核通过时预占库存”规则已停用，不再作为 V5 开发、测试、验收依据。 |
| BR-012 | 报表金额权限 | 生效 | 财务报表不受金额限制；仓库导出不能显示金额。 |
| BR-013 | 报表运营公司列 | 生效 | 销售报表必须包含“运营公司”列，列值取运营审核人的公司名称。 |
| BR-014 | 订单进度显示 | 生效 | 已审批节点显示处理用户名，未审批节点不显示。 |
| BR-015 | 首页待审数量 | 生效 | 小程序首页“待审订单”入口必须显示当前角色待审核数量。 |
| BR-016 | 历史文档效力 | 生效 | 已归档文档不得作为当前版本需求、开发、测试和验收依据。 |
| BR-017 | 激活码字符集 | 生效 | 业务用户激活码仅允许字母和数字，不允许包含 `-`。 |
| BR-018 | 报表中心角色开放与隔离 | 生效 | 小程序报表中心向客户、运营、财务、供应商、仓库、管理员开放，并按角色隔离报表范围与字段。 |
| BR-019 | 收货单位名称隔离 | 生效 | 供应商与仓库在待审订单和订单详情中不得看到收货单位名称。 |
| BR-020 | 后台管理菜单分组 | 生效 | Web 后台当前按“公司主体、用户管理、模板中心、销售合同、采购合同、销售订单、采购订单、采购入库、仓库管理、库存调整、库存台账、报表中心、审计日志”分组，激活链接并入用户管理。 |
| BR-021 | 协议模板中心 | 生效 | 后台已提供协议模板菜单，统一管理发货指令单、采购合同、销售合同模板；当前每类模板仅维护一份生效模板正文，不提供版本管理。 |
| BR-022 | 采购合同与采购入库 | 生效 | 后台已提供采购合同与采购入库能力；采购合同生效后自动生成采购入库单，采购入库确认后才增加库存；采购订单不触发采购入库。 |
| BR-023 | 销售合同管理 | 生效 | 后台已提供销售合同能力，销售合同支持模板选择、自动计算金额与合同 PDF 一键生成下载。 |
| BR-024 | 双订单重构 | 生效 | 当前订单模型已重构为销售订单和采购订单，两者保持关联但独立流转。 |
| BR-025 | 合同多订单复用与超执行 | 生效 | 一份销售合同或采购合同可对应多张订单，最后一张订单允许超过合同当前待执行数量；当前版本不生成补差子订单。 |
| BR-026 | 销售合同保证金与生效条件 | 生效 | 销售合同包含保证金字段，默认10%；销售合同生效前必须上传盖章销售合同和客户保证金银行回单。 |
| BR-027 | 采购合同保证金与生效条件 | 生效 | 采购合同包含保证金字段，默认10%；采购合同生效前必须上传盖章采购合同和付给供应商保证金的银行回单。 |
| BR-028 | 销售订单合同关联与金额快照 | 生效 | 客户销售订单必须关联已生效销售合同，订单金额按销售合同快照确定，且允许订单数量大于合同当前待执行数量。 |
| BR-029 | 采购订单合同关联、自动选合同与付款凭证 | 生效 | 销售订单转采购订单时必须关联已生效采购合同；采购订单数量不允许修改，直接沿用销售订单数量；系统默认自动选择同油品下未执行完数量最少的采购合同，若并列则按生效日期最早、合同号最小优先；自动选择时必须展示合同关键信息并二次确认，允许手工改选；若无可用采购合同，则自动生成“待提交”采购订单，后续可人工补录合同并提交；采购订单必须上传“向供应商付款的付款凭证”后才能向下流转；采购订单提交时必须人工选择发货指令单模板，系统按所选模板自动生成发货指令单 PDF，并内嵌在采购订单详情中。 |
| BR-030 | 订单完成、合同执行与库存扣减口径 | 生效 | 销售订单经运营审核通过、进入财务审批节点时冻结库存；采购订单进入 `WAREHOUSE_PENDING` 时销售订单同步进入 `READY_FOR_OUTBOUND`；销售订单完成标志为一次实际出库数量不得低于订单数量且不得超过订单数量的 `110%`；采购订单完成标志为对应销售订单完成；销售合同、采购合同的可执行数量和库存扣减统一按实际出库数量扣减，报表同时展示待执行数量与超执行数量。 |
| BR-031 | 合同创建角色与审批边界 | 生效 | 合同仅允许财务、运营、业务管理员创建和维护；当前版本不开发合同审批流，但保留完整操作日志与生效附件；合同生效后不允许修改。 |
| BR-032 | 金额、吨位与凭证校验口径 | 生效 | 保证金按含税金额口径计算；金额统一保留2位小数，吨位统一保留4位小数，采用四舍五入；客户收款凭证、保证金回单、向供应商付款凭证仅校验上传与存在，不做系统内金额匹配或合规校验。 |
| BR-033 | 后台分域报表中心 | 生效 | 后台报表中心已按销售订单报表、采购订单报表、销售合同报表、采购合同报表、库存变动报表、仓库出入库台账分域实现。 |
| BR-034 | 出库次数与库存冻结规则 | 生效 | 一张销售订单仅允许一次出库，不允许部分出库或多次出库；冻结库存不计入可用库存；仓库出库前必须校验可用库存，不足则禁止完成出库；当前版本不新增计量误差容差处理。 |
| BR-035 | 采购入库与库存调整单 | 生效 | 采购入库必须绑定采购合同，并由采购合同生效后自动生成采购入库单；采购订单不产生采购入库；历史库存初始化和特殊情况必须走独立库存调整单。 |
| BR-036 | 报表角色范围与仓库台账 | 生效 | 客户仅可查看销售相关报表；供应商仅可查看采购类报表；仓库仅可查看仓库出入库台账，台账只展示数量和相关采购订单信息，不展示金额；运营、财务、业务管理员、超级管理员可查看全部后台报表；各类报表必须按角色裁剪字段。 |
| BR-037 | 合同补差机制 | 规划中 | 一期不生成补差子订单，改为手工补差合同：销售补差合同必须关联原销售合同，仅用于增加原销售合同可执行数量；采购补差合同必须关联原采购合同，自动生成采购补差入库单，入库后增加库存和原采购合同可执行数量；补差合同不得改绑历史订单。 |
| BR-038 | 双订单异常关闭与资源释放 | 生效 | 财务收款后如采购执行失败或采购合同确认错误并已继续流转，则销售订单与采购订单直接异常关闭，释放冻结库存及相关执行资源；系统不支持原单重开，需重新下单；客户收款确认后的销售订单与全部采购订单不再提供驳回，只允许业务管理员执行异常关闭。 |
| BR-039 | 运输资料快照维护边界 | 生效 | 运输资料仅允许客户维护，保留历史运输信息一键带入能力；提交销售订单即锁定；运营、财务、供应商、仓库仅可查看快照；当前版本不做实名与车辆合规校验。 |
| BR-040 | 合同作废与关闭边界 | 生效 | 合同生效后不允许修改；合同关联合同订单、采购入库、仓库出库等任何业务单据后不得作废，不支持手工关闭，特殊情况线下处理。 |
| BR-041 | 销售订单与采购订单一一对应 | 生效 | 一张销售订单仅对应一张采购订单，当前版本不预留拆分为多张采购订单的业务接口。 |
| BR-042 | 订单驳回与异常关闭权限边界 | 生效 | V5 中运营、财务、供应商、仓库不再具备驳回或异常关闭权限，仅业务管理员可执行订单终止类动作；销售订单仅在客户收款确认前允许驳回，客户收款确认后及全部采购订单仅允许异常关闭；其他角色仅保留正向通过、提交、确认、出库完成等动作。 |
| BR-043 | V5 上线数据边界 | 生效 | V5 作为全新系统一次性上线，不做 V4 历史订单、合同、库存、报表数据迁移；上线后业务数据仅以 V5 新录入和初始化库存调整数据为准。 |
| BR-044 | V5 小程序实施方式 | 生效 | V5 小程序按全新目录重建，不在 V4 单订单页面结构上继续兼容改造；原小程序源码归档，仅保留追溯价值。 |
| BR-045 | V5 终端承载边界 | 生效 | 小程序仅承接订单执行链、合同与报表轻量查询；后台承接模板中心、主数据、合同维护与生效作废、采购入库确认、库存调整、审计与重报表管理。仓库主体前台仅保留出库执行和仓库报表查看。 |
| BR-046 | 模板中心权限与审计边界 | 生效 | 协议模板中心仅允许 `SUPER_ADMIN` 在 Web 后台维护；普通业务管理员不得访问模板 CRUD；模板新增、修改、启停、设默认等动作必须记录真实超级管理员身份，不得仅记录桥接账号。 |
| BR-047 | 仓库归属锁定规则 | 生效 | 仓库仅允许绑定启用中的仓库主体公司；仓库一旦关联合同执行、采购订单、采购入库、库存流水或库存调整等业务数据，不允许改绑公司或解绑，避免历史采购、出库与台账可见范围被重写。 |
| BR-048 | 用户角色与公司绑定规则 | 生效 | `CUSTOMER` 必须绑定启用中的客户公司，`SUPPLIER` 必须绑定启用中的供应商公司，`OPERATOR` 必须绑定启用中的运营公司，`WAREHOUSE` 必须绑定启用中的仓库主体公司；`FINANCE`、`ADMIN` 可不绑定公司，若绑定则仅允许启用中的运营公司。 |
| BR-049 | 公司类型锁定规则 | 生效 | 公司主体一旦被用户、合同、订单、仓库或库存业务引用，不允许在线修改公司类型；确需调整时应新建主体并线下完成切换，不得通过修改历史主体类型改写业务语义。 |
| BR-050 | 运营公司与仓库主体职责边界 | 生效 | 运营公司是系统主经营主体，承接运营审核、财务管理、业务管理员归属、合同/订单管理、采购入库、库存调整、报表与审计视角；仓库主体是相对运营公司的第三方协同主体，地位与客户、供应商相同，仅承接仓库归属、仓库账号归属、出库执行和仓库数量型报表查看。仓库主体不得作为系统主经营主体使用，也不得承接财务、管理员和合同管理归属。 |
| BR-051 | V4 代码、接口与测试归档边界 | 生效 | V4 单订单系统代码、接口、测试脚本与旧自动化方案统一归档，不再在当前应用入口注册、不再参与测试、不再作为 V5 交付内容；V5 当前仅承接销售订单、采购订单、合同、库存、报表与后台管理域。 |
| BR-052 | 小程序全自动 UAT 技术路线 | 生效 | 小程序全自动 UAT 采用自研测试框架：由启动页在本地测试环境主动拉取 UAT 计划、自行执行订单主链并回传结果；后台编排脚本仅负责清库、生成基线上下文、下发计划、拉起微信开发者工具和汇总结果，不再依赖 `Minium`、`miniprogram-automator` 或系统文件选择器。 |
| BR-053 | 测试环境快捷登录与激活口径 | 弃用 | 曾允许在非生产环境使用 `root / 1 / 1` 登录管理后台，并将小程序业务用户激活码对外统一展示为 `1`；该口径因安全风险和误用风险过高而停用，不得再作为测试、联调和验收依据。 |
| BR-054 | 测试环境默认模板基线 | 生效 | 仅在非生产环境允许通过测试初始化脚本写入默认测试模板，包含发货指令单、销售合同、采购合同三类模板示例正文；模板仅用于联调与手工测试，不得作为正式生产合同正文依据。 |
| BR-055 | 测试环境清库保留范围 | 生效 | 测试环境清库时必须保留超级管理员账户、模板桥接账号以及全部协议模板配置；其他业务账号、公司、合同、订单、库存、报表、日志可清空。 |
| BR-056 | 合同模板自动带出录入骨架 | 生效 | Web 后台新建销售合同、采购合同时，必须先完成合同号、合同日期、交易对手、仓库（采购合同）以及至少一条完整合同条目录入后，才允许选择合同模板；模板选定后，系统必须基于已选公司、仓库和合同条目自动生成合同正文预览、动态字段快照和合同条目展示，不再允许业务人员手工填写 `variable_snapshot JSON` 或逐项补录动态字段。 |
| BR-057 | 测试环境认证与激活恢复正式规则 | 弃用 | 曾要求测试环境超级管理员登录恢复真实密码校验和真实 TOTP 校验；该口径已停用，不再作为当前测试、联调、验收依据。 |
| BR-058 | 合同后台工作台布局 | 生效 | Web 后台合同管理按“合同列表、合同起草、合同签订”二级工作台拆分处理；列表页负责筛选、定位和直接下载合同 PDF，起草页负责合同基本信息、模板选择、自动带出的关键信息概况展示和合同条目录入，签订页负责下载合同、上传签订附件、提交生效和作废，避免在同一页面同时堆叠查询、录入和生效动作。 |

| BR-095 | 小程序订单、合同、报表固定时间筛选口径 | 生效 | 小程序订单列表、合同中心、报表中心统一仅提供“今天、本周、本月、今年、全部”五档固定时间筛选，默认选中“本周”；仓库角色“待出库订单”入口仅展示 `WAREHOUSE_PENDING` 状态单据且不展示状态筛选，避免仓库执行页混入非待出库节点单据。 |

## 4. 业务目标与场景
- 目标：支撑成品油批发业务的低频、低并发订单流转，重点保障可执行、可审计、可追溯。
- 核心流程：客户下单 -> 运营审核 -> 财务审核 -> 供应商审核 -> 仓库出货 -> 订单结束。
- 管理模式：`SUPER_ADMIN` 仅在 Web 后台管理配置与账号；业务角色主要在微信小程序执行。
- [生效] 终端分工：小程序作为订单执行前台，承接客户下单、审批、上传附件、进度查看与轻量报表下载；后台作为管理中台，承接模板、主数据、合同维护、生效作废、采购入库、库存调整和审计管理。

## 5. 角色与权限
- [生效] `SUPER_ADMIN`：仅 Web 登录；管理系统配置、模板、用户、主数据、审计日志。
- [生效] 小程序角色边界：除 `SUPER_ADMIN` 外的业务角色仅在小程序执行订单链和轻量查询，不在小程序维护模板、主数据、合同生效作废、采购入库与库存调整。
- [生效] Web 后台菜单收口：激活链接并入“用户管理”；当前后台按公司主体、用户管理、模板中心、销售合同、采购合同、销售订单、采购订单、采购入库、仓库管理、库存调整、库存台账、报表中心、审计日志分组。
- [生效] 业务用户激活码：仅允许字母和数字组成，不允许包含 `-`。
- [生效] 模板中心权限：模板中心只允许 `SUPER_ADMIN` 进入和调用接口；普通 `ADMIN` 仅可执行订单终止、查看订单与日志，不可维护模板。
- [生效] `ADMIN`（业务管理员）：小程序仅查看待审订单与执行订单终止类动作，不参与运营/财务/供应商/仓库审批；可查看历史订单列表。
- [生效] `CUSTOMER`：仅可创建订单、查看自己订单与审批状态。
- [生效] 客户前台文案：在小程序客户角色视角中，“销售订单/销售合同”统一简化展示为“订单/合同”。
- [生效] 客户首页入口文案：小程序客户角色首页中的销售订单入口统一展示为“订单中心”，销售合同入口统一展示为“合同中心”，销售报表入口统一展示为“报表中心”。
- [生效] 前后台状态文案：小程序与后台中的业务状态、流程状态、日志状态、报表导出状态统一展示中文，不直接显示英文枚举值。
- [生效] `OPERATOR`：审核客户订单，不再填写销售单价。
- [生效] 运营工作台入口：小程序 `OPERATOR` 角色工作台统一展示“待审订单”“订单中心”“合同中心”“报表中心”等经营入口，不再拆分单独的“销售合同”“采购合同”双入口；其中“待审订单”承接待运营审核订单列表，“订单中心”复用销售订单列表页但不展示“新建订单”按钮。
- [生效] `FINANCE`：审核收款，填写收款金额并上传客户收款凭证；提交采购订单时必须同步上传向供应商付款凭证，提交后直接流转到供应商审核节点（BR-078）。
- [生效] 财务工作台入口：小程序 `FINANCE` 角色工作台统一展示“待审核销售订单”“待处理采购订单”“订单中心”“合同中心”“报表中心”；其中“待审核销售订单”仅展示 `OPERATOR_APPROVED` 状态销售订单，“待处理采购订单”仅展示 `PENDING_SUBMIT` 状态采购订单，且两个待办页面均不展示筛选条件；“订单中心”提供销售订单、采购订单两个查询入口。
- [生效] 财务与管理员工作台入口文案：小程序 `FINANCE`、`ADMIN` 角色工作台中的合同入口统一展示为“合同中心”，与运营工作台保持一致。
- [生效] `SUPPLIER`：审核采购订单中的系统发货指令单，必须上传盖章后的发货指令单。
- [生效] `WAREHOUSE`：仅执行仓库出货，上传出库单，并查看本仓库主体范围内的数量型仓库报表；工作台仅保留“待出库订单”和“仓库台账”入口，不参与财务、合同、采购入库、库存调整等管理动作。
- [生效] 小程序固定时间筛选：订单列表、合同中心、报表中心统一使用“今天、本周、本月、今年、全部”五档时间筛选，默认“本周”。
- [生效] 合同查询边界：小程序合同中心仅展示各角色权限范围内的合同摘要与执行情况，不再提供合同 PDF 打开动作；合同新建、修改、生效、作废仍只在后台执行。
- [生效] 出库单可见性：仓库上传出库单后，除 `CUSTOMER` 外的角色可在订单详情查看出库单。
- [生效] 用户公司绑定边界：后台创建或修改用户时必须校验角色与公司类型一致，且公司处于启用状态；不满足时禁止保存。
- [生效] 公司类型职责边界：运营公司是系统主经营主体，负责经营管理、订单审核、合同维护、采购入库、库存调整、报表分析和财务/管理员账号归属；仓库主体是第三方仓储协同主体，仅负责仓库归属、仓库账号归属、出库执行和仓库报表查看。

## 6. 订单状态机（当前线上基线）
### 6.1 销售订单状态
- [生效] `SUBMITTED`：客户已提交，待运营审核。
- [生效] `OPERATOR_APPROVED`：运营审核通过，待财务审核。
- [生效] `CUSTOMER_PAYMENT_CONFIRMED`：财务已确认客户收款，采购执行中。
- [生效] `READY_FOR_OUTBOUND`：关联采购订单已进入仓库待出库。
- [生效] `COMPLETED`：销售订单已完成（终态）。
- [生效] `REJECTED`：销售订单在客户收款确认前被业务管理员驳回（终态）。
- [生效] `ABNORMAL_CLOSED`：销售订单异常关闭（业务管理员终态）。

### 6.2 采购订单状态
- [生效] `PENDING_SUBMIT`：待财务提交采购订单。
- [弃用] `SUPPLIER_PAYMENT_PENDING`：原为"待财务上传向供应商付款凭证"独立节点；自 BR-078 生效后，付款凭证随采购订单提交同步上传，新单不再进入此状态；枚举保留用于历史数据兼容。
- [生效] `SUPPLIER_REVIEW_PENDING`：待供应商确认。
- [生效] `WAREHOUSE_PENDING`：供应商已确认，待仓库出库。
- [生效] `COMPLETED`：采购订单已完成（终态）。
- [生效] `ABNORMAL_CLOSED`：采购订单异常关闭（业务管理员终态）。

## 7. 驳回与终止规则
- [弃用] V4 旧版驳回统一为“驳回即终止”，不再回退到上一节点；该口径不再作为 V5 开发依据。
- [弃用] V4 旧版运营/财务/供应商/仓库驳回均必须填写原因并进行二次确认；V5 不再沿用该权限模型。
- [生效] V5 中订单终止类动作仅业务管理员可执行；销售订单仅在客户收款确认前允许驳回，客户收款确认后及全部采购订单仅允许异常关闭。
- [弃用] 客户撤销能力取消，旧版“客户可撤销订单”规则不再生效。

## 8. 订单字段与录入规则
### 8.1 订单基础信息
- [生效] 计划编号：系统自动生成。
- [生效] 订单日期：必填，可选未来日期。
- [生效] 收货单位：系统自动带出当前客户显示名称，不可修改。
- [生效] 小程序客户建单字段顺序：订单日期后，依次展示合同、油品、仓库、数量，再进入运输信息录入。
- [生效] 仓库名称：下拉选择（不要求用户感知仓库ID）。
- [生效] 油品：下拉选择，并与仓库联动。
- [生效] 数量(吨)：必填，>0，最多4位小数。
- [生效] 当前可用库存提示：系统配置开关控制，默认开启；客户端默认不显示。

### 8.2 运输信息（提交订单前必填）
- [生效] 小程序客户建销售订单页中，“运输快照”统一更名为“运输信息”。
- [生效] 必填：运输单位、司机姓名、手机号、身份证、车牌号；所有必填项必须显式展示必填标识。
- [生效] 可选：单双枪、是否带泵、核定载重（吨，正整数）、备注。
- [生效] 单双枪：采用“单枪 / 双枪”二选一，不允许自由手填。
- [生效] 是否带泵：采用“带泵 / 不带泵”二选一，默认“不带泵”。
- [生效] 校验：
  - 手工字段不能含空格。
  - 司机姓名1-4个汉字。
  - 手机号必须为中国手机号。
  - 身份证必须合规。
  - 车牌必须为中国车牌，自动转大写。
- [生效] 历史带出：支持历史运输信息一键带入。
- [生效] 历史沉淀与删除：客户成功提交订单后，当前运输信息必须自动沉淀到历史运输信息池；若与历史池已存在记录内容完全一致，则复用原记录并刷新最近使用时间，不重复新增；历史列表必须支持客户删除不再使用的运输信息，删除仅影响后续预选，不得改写历史订单上的运输快照。

### 8.3 附件规则
- [生效] 客户建销售订单页附件区统一按“付款凭证”展示，支持图片/PDF/Excel，至少上传1个文件后才允许提交销售订单。
- [生效] 上传限制（当前实现口径）：单次文件选择为1个；服务端单文件大小<=20MB；销售订单付款凭证附件总数最多20个（由建单入参校验控制）。
- [生效] 受保护文件打开：小程序报表中心、销售订单详情、采购订单详情等所有通过受保护文件接口读取附件的页面，必须统一按“标准文件名响应头 + 文件名/响应头联合识别文件类型”的方式打开；图片类附件走图片预览，PDF/Excel 等文档附件走文档打开，不得仅依赖下载 URL 后缀判断。

## 9. 审核节点业务规则
### 9.1 运营审核
- [生效] 审核通过：直接审核通过，不再填写销售单价，不再生成应收金额。
- [弃用] V4 旧版运营审核驳回：必填原因 + 二次确认；V5 不再提供运营驳回动作。

### 9.2 财务审核
- [生效] 审核通过：必填收款金额 + 客户收款凭证。
- [生效] 审核提示：不再处理应收金额比对，仅按实际收款金额与客户收款凭证流转。
- [生效] 审核通过后：系统自动生成关联采购订单，不在财务审核节点直接生成发货指令单 PDF。
- [生效] 发货指令单：由财务在采购订单提交时人工选择模板，系统按所选模板自动生成 PDF；模板由 SUPER_ADMIN 在后台维护标题、发货单位、正文模板、页脚与字体大小。
- [生效] 财务提交采购订单成功后，采购订单详情必须立即返回并展示系统生成的发货指令单 PDF；财务、运营、管理员、供应商在各自权限范围内都应可直接打开发货指令单，仓库仍仅查看供应商上传的盖章发货指令单。
- [生效] 采购付款凭证：财务提交采购订单时必须同步上传向供应商付款凭证（支持1-9个文件），随提交操作一同完成，提交后直接进入供应商审核节点（BR-078）。
- [弃用] V4 旧版财务审核驳回：必填原因 + 二次确认；V5 不再提供财务驳回动作。

### 9.3 供应商审核
- [生效] 审核通过：必须上传盖章发货指令单。
- [生效] 审核资料来源：供应商先查看采购订单详情中系统生成的发货指令单 PDF，盖章后回传附件。
- [弃用] V4 旧版供应商审核驳回：必填原因 + 二次确认；V5 不再提供供应商驳回动作。

### 9.4 仓库出货
- [生效] 出货完成：必填出库单，提交后订单结束。
- [弃用] V4 旧版仓库出货驳回：必填原因 + 二次确认；V5 不再提供仓库驳回动作。

## 10. 数据可见性与隔离
- [生效] 客户：仅查看自己提交订单；仅看客户侧信息和审批状态，不看运营/财务新增信息，且不展示出库单查看入口。
- [生效] 供应商：仅看运输执行资料与供应商执行附件，不看金额与客户收款凭证；待审订单与订单详情不展示收货单位名称；可查看采购订单详情中的发货指令单 PDF 并上传盖章发货指令单。
- [生效] 仓库：仅看运输执行资料 + 供应商上传的盖章发货指令单；不看运营/财务新增金额信息；待审订单与订单详情不展示收货单位名称；可查看仓库上传的出库单。
- [弃用] 仓库直接以财务节点原始提货指令单作为执行附件的旧口径已停用。
- [生效] 运营/财务/管理员：可查看全量订单（客户信息隔离仅对客户生效），可查看供应商盖章单与仓库上传的出库单。

## 11. 看板与列表规则
- [生效] 各角色工作台“待审订单”仅展示当前流转到本角色的订单。
- [生效] 小程序首页“待审订单”入口展示当前角色待审核数量。
- [生效] 小程序销售订单待审列表进入时，默认按当前角色落在可处理状态：运营默认“待运营”，财务默认“待财务”；用户仍可手工切换到其他状态查看历史订单。
- [生效] 订单列表支持查看其他状态历史订单。
- [生效] 小程序合同中心仅展示执行中的合同，按合同日期由近到远排序，单个标签页最多展示 10 条；其他状态合同不在列表中展开，页面需明确提示用户。
- [生效] 运营工作台库存概览：小程序 `OPERATOR` 角色工作台必须与财务工作台一致展示库存概览，统一按仓库展开油品库存、预占和可用数量，服务成品油批发运营、油库管理和供应链协同调度。
- [生效] 小程序合同中心动作边界：所有可访问合同中心的角色在销售合同、采购合同列表中均不展示“查看合同 PDF”按钮，仅保留合同摘要、执行进度及仓库等关键信息查看。
- [生效] 运营、财务、管理员工作台合同入口命名统一为“合同中心”，避免相同合同域在不同角色工作台中出现“中心/查询”双口径并存。
- [生效] Web 后台合同列表按合同条目展开：销售合同、采购合同列表必须增加“合同数量（吨）”列，并按合同条目逐行展示；同一合同有多个条目时，油品与合同数量按条目展开，合同号、交易对方、状态、待执行量/待入库量与操作等合同级字段合并展示，便于成品油经营协同、仓储衔接与审计复核。
- [生效] 订单列表支持分页加载与时间窗口筛选（近7/30/90/365天），避免历史数据累积导致页面过长或卡顿。
