# V6 需求闭卷核验题库（阶段C前）

## 1. 使用方式
- 用途：验证对关键业务规则、口径与边界的掌握程度。
- 执行方式：闭卷问答（口头或书面），每题必须给出“规则号+结论+关键条件”。
- 判定标准：
  - 总分100分，建议通过线 `>=95分`。
  - 阻断题（标注`阻断`）必须全对，否则不通过。

## 2. 题目与标准答案

1. `阻断` 采购订单付款为0是否一律要走阈值校验？
标准答案：销售订单衍生采购订单场景下，按规则11无条件放行，不校验规则14阈值。

2. `阻断` 规则11与规则14冲突时谁优先？
标准答案：规则11优先；规则14不适用于规则11定义场景。

3. `阻断` 销售合同与采购合同金额闭环是否可以混算？
标准答案：不可以，必须按合同方向分开闭环核算。

4. 订单单价来源于哪里？实收实付来源于哪里？
标准答案：单价来源合同单价；实收实付来源财务确认的收付款单金额。

5. 附件应挂在订单还是收付款单？
标准答案：按单据类型归属，不一刀切。合同附件归合同，订单业务附件归订单，付款凭证归付款单，收款凭证归收款单，发货指令单附件归采购订单。

6. 采购合同生效后系统自动生成哪两类单据？
标准答案：付款单（保证金）与入库单（草稿）。

7. 入库单草稿何时计入库存与履约数量？
标准答案：录入实际数量并提交生效后才计入。

8. 销售合同生效后系统自动生成什么？
标准答案：收款单（保证金）。

9. 销售订单财务审批通过后系统自动生成什么？
标准答案：采购订单、付款单（采购实付）、收款单（销售实收）。

10. 销售合同对应出库量由哪两部分构成？
标准答案：仓库正常流程出库单 + 手工补录出库单。

11. 合同履约数量上限如何控制？
标准答案：按 `0.001` 数量精度计算后，累计实际履约数量不得超过合同数量*超量履约阈值，超限阻断单据生效。

12. 双阈值分别是什么？
标准答案：保证金放行阈值、合同超量履约阈值；二者为系统级统一参数下发，且前者不得大于后者。

13. 收款单0金额但不满足规则14时如何处理？
标准答案：转待补录金额并阻断提交。

14. 手工补录付款单的强制关联条件是什么？
标准答案：必须同时关联采购合同与采购订单。

15. 手工补录出库单的强制关联条件是什么？
标准答案：必须绑定销售合同。

16. 合同自动关闭需要满足哪两个条件？
标准答案：数量履约完成 + 金额闭环满足（金额差额绝对值不超过 `0.01`）。

17. 合同手工关闭最少应具备哪些控制？
标准答案：原因必填、二次确认、审计留痕、按五步顺序处理。

18. 单据上下游关系最低要求是什么？
标准答案：合同/订单/收付款/出入库均记录上下游ID，支持一对多与多对多。

19. 小程序是否允许执行审批驳回并填写原因？
标准答案：不允许，仅查看驳回结果。

20. 管理后台首版仪表盘四个固定指标是什么？
标准答案：合同执行率、当日实收实付、库存周转、超阈值告警数。

21. 当前版本并发策略是什么？
标准答案：低并发假设，幂等+事务，不引入分布式锁与高并发强一致。

22. 当前版本是否建设多活与跨地域容灾？
标准答案：不建设，属于非目标。

23. 当前版本是否建设完整财务核算（总账/成本/税务）？
标准答案：不建设，系统仅做业务链与收付款辅助证据链。

24. 报表更新机制采用什么方式？
标准答案：事件触发增量更新 + 定时全量校准。

25. 报表口径变更时能否直接覆盖旧口径？
标准答案：不能；需版本化并保留快照，可追溯。

## 3. 评分模板

| 类别 | 题号 | 分值 | 得分 | 备注 |
|---|---|---|---|---|
| 阻断题 | 1,2,3 | 15 |  | 必须全对 |
| 资金与合同口径 | 4~15 | 45 |  |  |
| 关闭与追溯 | 16~18 | 15 |  |  |
| 端边界与报表 | 19,20,24,25 | 15 |  |  |
| 范围边界 | 21~23 | 10 |  |  |

## 4. 通过判定
- 通过：总分 `>=95` 且阻断题全对。
- 不通过：任一阻断题错误，或总分 `<95`。

## 5. 复测规则
- 首次不通过时，需先对错题对应规则进行复盘，再进行二次闭卷。
- 二次闭卷仍不通过，则暂停开发，回到需求澄清会。
