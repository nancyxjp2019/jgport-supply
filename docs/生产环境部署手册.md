## 4. 执行命令（超级管理员保留）

```bash

JWT_SECRET=$(grep '^jwt_secret_key=' /data/jgsd/app/project/backend/.env | sed 's/^jwt_secret_key=//')
WECHAT_APP_ID="替换为你的小程序AppID"
WECHAT_APP_SECRET="替换为你的小程序AppSecret"
OSS_ACCESS_KEY_ID="替换为你的OSS AccessKey ID"
OSS_ACCESS_KEY_SECRET="替换为你的OSS AccessKey Secret"

sudo bash deploy/alinux_one_click_deploy.sh \
  --repo-url "git@github.com:nancyxjp2019/jgsd.git" \
  --domain "sd.jgport.top" \
  --jwt-secret "$JWT_SECRET" \
  --init-super-admin false \
  --app-user root \
  --conda-home /root/miniconda3 \
  --app-root /data/jgsd \
  --wechat-login-mode "official" \
  --wechat-app-id "$WECHAT_APP_ID" \
  --wechat-app-secret "$WECHAT_APP_SECRET" \
  --file-storage-backend "oss" \
  --oss-bucket-name "jgport" \
  --oss-endpoint "oss-cn-shenzhen.aliyuncs.com" \
  --oss-access-key-id "$OSS_ACCESS_KEY_ID" \
  --oss-access-key-secret "$OSS_ACCESS_KEY_SECRET" \
  --oss-prefix "media"


```

> 安全提醒：严禁将真实 `AppSecret`、OSS 密钥写入 Git 仓库；若曾泄露，请立即在微信公众平台与阿里云控制台完成密钥轮换。

## 5.1 中断重试说明（已支持）

- 若首次部署在“拉取代码”阶段异常中断，导致 `/data/jgsd/app/project` 已存在但不是完整 Git 仓库：

## 6. 常用参数

- `--repo-branch main`
- `--init-super-admin false`
- `--enable-ssl true`（会尝试使用 certbot 自动签发）
- `--wechat-login-mode mock|official`
- `--wechat-app-id xxx`
- `--wechat-app-secret xxx`
- `--wechat-api-timeout 8`
- `--activation-code-expire-minutes 1440`（激活码默认有效 24 小时）
- `--file-storage-backend local|oss`
- `--oss-bucket-name xxx`
- `--oss-endpoint xxx`
- `--oss-access-key-id xxx`
- `--oss-access-key-secret xxx`
- `--oss-base-url xxx`（可选）
- `--oss-prefix attachments`

### 5.1 微信登录参数说明（生产必看）

- `mock`：仅用于本地联调，不调用微信官方接口。
- `official`：调用微信 `code2session` 获取真实 `openid`，生产环境必须使用。
- 当 `--wechat-login-mode official` 时，`--wechat-app-id` 与 `--wechat-app-secret` 必填。
- 脚本会写入 `backend/.env`：
  - `wechat_login_mode`
  - `wechat_app_id`
  - `wechat_app_secret`
  - `wechat_api_timeout_seconds`

### 5.2 OSS 参数说明（附件存储）

- `--file-storage-backend oss` 时，附件与发货指令单 PDF 会存储到 OSS。
- 推荐使用私有桶，并保持 `--oss-base-url` 为空：
  - 文件访问走后端代理地址 `/api/v1/files/object/{file_key}`。
  - 小程序无需额外配置 OSS 域名白名单。
- 若使用公共读或 CDN 域名，可设置 `--oss-base-url`，系统将直接返回该域名文件地址。

## 6.2 公网限流建议（Nginx）

- 作用：对登录、激活绑定、上传等高风险接口做第一层限流，降低撞库和刷接口风险。
- 说明：以下示例需放在 Nginx `http` 与站点 `server` 配置中，并按实际并发微调阈值。

```nginx
# http 级别
limit_req_zone $binary_remote_addr zone=jgsd_auth_zone:10m rate=20r/m;
limit_req_zone $binary_remote_addr zone=jgsd_wechat_zone:10m rate=60r/m;
limit_req_zone $binary_remote_addr zone=jgsd_upload_zone:10m rate=40r/m;

server {
    listen 443 ssl http2;
    server_name sd.jgport.top;

    location = /api/v1/admin/auth/login {
        limit_req zone=jgsd_auth_zone burst=10 nodelay;
        proxy_pass http://127.0.0.1:8000;
    }

    location = /api/v1/auth/wechat/login {
        limit_req zone=jgsd_wechat_zone burst=20 nodelay;
        proxy_pass http://127.0.0.1:8000;
    }

    location = /api/v1/auth/wechat/bind {
        limit_req zone=jgsd_wechat_zone burst=10 nodelay;
        proxy_pass http://127.0.0.1:8000;
    }

    location = /api/v1/files/upload {
        limit_req zone=jgsd_upload_zone burst=20 nodelay;
        proxy_pass http://127.0.0.1:8000;
    }
}
```

## 6.3 一键应用/回滚限流（推荐）

- 脚本路径：`deploy/nginx_rate_limit_guard.sh`
- 功能：
  - `apply`：自动备份并注入限流规则。
  - `rollback`：自动移除限流规则并回滚。
  - `status`：查看当前是否已启用。

```bash
# 1) 查看状态
sudo bash deploy/nginx_rate_limit_guard.sh status

# 2) 一键启用限流
sudo bash deploy/nginx_rate_limit_guard.sh apply

# 3) 验证配置
sudo nginx -t
sudo systemctl status nginx --no-pager

# 4) 一键回滚（如需）
sudo bash deploy/nginx_rate_limit_guard.sh rollback
```

- 备份目录：`/etc/nginx/conf.d/jgsd_rate_limit_backups`
- 默认修改文件：
  - 站点配置：`/etc/nginx/conf.d/jgsd.conf`
  - 限流 zone：`/etc/nginx/conf.d/jgsd_rate_limit_zones.conf`

## 8. 数据备份（PostgreSQL 18）

- 策略：
  - 每天 `23:00` 自动执行一次 `pg_dump` 逻辑备份
  - 仅保留最近 `7` 份备份，超过后自动删除最早备份

```bash
# 1) 手动执行一次备份（先验证权限）
export PGPASSWORD='请替换为数据库密码'
pg_dump -h 127.0.0.1 -p 5432 -U jgport -d jgport_prod -Fc -f /backup/jgport_$(date +%F_%H%M%S).dump

# 2) 配置 crontab（每天 23:00 自动备份并清理旧备份）
crontab -e
0 23 * * * PGPASSWORD='请替换为数据库密码' pg_dump -h 127.0.0.1 -p 5432 -U jgport -d jgport_prod -Fc -f /backup/jgport_$(date +\%F_\%H\%M\%S).dump && ls -1t /backup/jgport_*.dump | tail -n +8 | xargs -r rm -f

# 3) 校验备份文件
ls -lah /backup/jgport_*.dump | head
```
