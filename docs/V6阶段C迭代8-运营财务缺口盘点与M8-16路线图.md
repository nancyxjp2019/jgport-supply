# V6 阶段C迭代8 运营财务缺口盘点与 M8-16+ 路线图

## 1. 结论先行
- 当前仓库并不存在“供应商功能明显多于运营/财务核心业务后端”的事实；`M3~M7` 已完成订单、资金、库存、合同关闭、报表等主链规则与接口。
- 当前真正不均衡的部分是：`M8` 对供应商小程序能力拆成了 `M8-12 ~ M8-15` 连续小任务包，而运营/财务后台页面仍大量停留在大模块文档或后端接口层，缺少等粒度的页面任务包与实现计划。
- 因此，后续 `M8-16` 起应优先补齐“后台页面能力欠账 + 文档颗粒度欠账”，而不是继续优先扩展供应商新动作。

## 2. 已实现能力盘点（运营/财务主链）

### 2.1 后端规则与接口已完成

| 模块 | 已完成能力 | 主要依据 |
|---|---|---|
| M3 订单域 | 销售订单创建、提交、运营审批、财务审批、销售转采购、采购订单详情 | `docs/V6阶段C迭代3-M3任务包.md` |
| M4 资金域 | 收款单/付款单草稿、手工补录、确认、凭证路径、规则11/14 | `docs/V6阶段C迭代4-M4任务包.md`、`backend/app/api/routers/funds.py` |
| M5 库存执行 | 入库单、出库单、仓库正常回执、手工补录、履约累计 | `docs/V6阶段C迭代5-M5任务包.md` |
| M6 合同关闭 | 自动关闭、手工关闭、关闭收口、关闭后阻断 | `docs/V6阶段C迭代6-M6任务包.md` |
| M7 报表看板 | 仪表盘、业务看板、轻量报表、快照留痕 | `docs/V6阶段C迭代7-M7任务包.md` |

### 2.2 M8 已完成的运营/财务端能力
- 管理后台首批：仪表盘、业务看板、登录与身份透传。
- 小程序运营/财务侧：轻量报表、待办入口、消息中心、消息回看。
- 以上能力的价值在于“可看见主链摘要”，但尚不足以替代真正的后台处理台。

## 3. 缺口盘点

### 3.1 页面级缺口（最主要）

| 领域 | 当前状态 | 缺口说明 |
|---|---|---|
| 管理后台订单页 | 未落地 | 后端已具备订单审批链路，但后台缺少订单列表、详情、运营审批、财务审批页 |
| 管理后台资金页 | 未落地 | 后端已有收付款接口，但缺少财务处理台、补录页、确认页、凭证回看页 |
| 管理后台库存页 | 未落地 | 后端已有出入库与执行链路，但缺少运营/管理侧追踪与异常处理页 |
| 管理后台合同关闭页 | 未落地 | 后端已有自动关闭与手工关闭，但缺少关闭入口、差异展示与关闭回看页 |
| 多维报表页 | 未落地 | `M7` 明确 `/reports/admin/multi-dim` 仍是 `501` |

### 3.2 业务能力缺口（后端/接口仍未补）

| 领域 | 当前状态 | 依据 |
|---|---|---|
| 退款/核销 | 未实现 | `docs/V6阶段C迭代4-M4任务包.md` |
| 收付款单驳回/待审核独立接口 | 未实现 | `docs/V6阶段C迭代4-M4任务包.md` |
| 每日闭环扫描/履约滞留扫描 | 未实现 | `docs/V6阶段C迭代6-M6任务包.md` |
| 多维管理报表与导出 | 未实现 | `docs/V6阶段C迭代7-M7任务包.md` |
| 按钮级细粒度权限矩阵 | 未实现 | `docs/V6阶段C迭代8-M8-03任务包.md` |

### 3.3 文档颗粒度缺口
- 供应商协同能力在 `M8-12 ~ M8-15` 被连续细拆，方便验收、回归与追溯。
- 运营/财务后台能力当前大多仍集中在 `M3/M4/M6/M7/M8-01` 等大包里，缺少“页面级任务包”。
- 这种颗粒度不一致会造成误判：看起来像是“供应商做得更多”，实际上常常是“后台同类能力没有拆文档”。

## 4. 原因判断
- 第一，`M8` 的主线是端能力落地，而供应商此前几乎没有真实业务页，因此被拆成连续小任务包逐步补齐。
- 第二，运营/财务的主业务规则已在 `M3~M7` 先完成后端主链，后续后台页面迁移没有同步按同等粒度拆包。
- 第三，`admin-web` 当前仅落地登录、仪表盘、业务看板三类页面，说明后台业务页面迁移尚未真正进入主线开发阶段。

## 5. M8-16+ 推荐路线

| 推荐顺序 | 任务包建议 | 目标 |
|---|---|---|
| M8-16（已完成） | 运营订单处理台首批 | 后台订单列表、详情、运营审批、财务审批入口 |
| M8-17（已完成） | 财务资金处理台首批 | 收款单/付款单列表、补录、确认、凭证回看 |
| M8-18 | 库存与执行跟踪台首批 | 入库/出库追踪、校验失败回看、执行异常提示 |
| M8-19 | 合同关闭与差异处理台首批 | 自动关闭结果回看、手工关闭入口、关闭差异展示 |
| M8-20 | 退款核销与资金驳回首批 | 退款/核销、待审核、驳回等后端与页面闭环 |
| M8-21 | 多维报表与导出首批 | 管理后台多维报表、筛选器、导出最小闭环 |

## 6. 近期执行建议
- 第一优先：`M8-18 库存与执行跟踪台首批`。
- 第二优先：`M8-19 合同关闭与差异处理台首批`。
- 第三优先：`M8-20 退款核销与资金驳回首批`。
- `M8-16` 与 `M8-17` 已完成，当前应继续承接 `M5/M6` 主链，补齐库存执行与合同关闭后台处理可见性。

## 7. 本文档用途
- 统一解释“为什么供应商文档显得更多”。
- 作为后续 `M8-16+` 任务包冻结顺序参考。
- 用于交付与评审时区分“后端已具备”与“页面未迁移”的真实差异。
