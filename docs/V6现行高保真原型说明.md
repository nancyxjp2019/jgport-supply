# V6 现行高保真原型说明

## 1. 文档定位
- 文档状态：`生效`
- 目标：为 `ADM-DASH-01`、`ADM-BOARD-01`、`MINI-REPORT-01` 及后续仍沿用同一视觉基线的页面提供高保真实现说明。
- 说明：本文档最初形成于 `M7`，当前继续作为 V6 现行 UI 实施基线使用。
- 设计依据：
  - `docs/需求方案.md`
  - `docs/V6阶段A-流程图状态机与UI原型清单.md`
  - `docs/V6阶段B-详细设计与原型冻结包.md`

## 2. 视觉总方向
- 管理后台风格：运营驾驶舱，不做花哨装饰，强调“异常优先、操作优先、数据优先”。
- 小程序风格：移动快捷卡片流，强调单手浏览、快速识别异常和入口清晰。
- 全站用户可见信息：统一中文。

## 3. 管理后台原型说明

## 3.1 ADM-DASH-01 经营仪表盘

### 3.1.1 页面结构
1. 顶部栏
   - 页面标题：`经营仪表盘`
   - 右侧信息：`口径版本 v1`、`更新时间 2026-03-06 14:25`、`SLA 正常/延迟`
2. 首屏四卡
   - 卡1：`合同执行率`
   - 卡2：`当日实收实付`
   - 卡3：`库存周转`
   - 卡4：`超阈值告警数`
3. 第二屏
   - 左：`异常分布`（待补录金额 / 校验失败 / 数量履约完成未关闭）
   - 右：`今日执行摘要`（实收、实付、入库量、出库量）
4. 第三屏
   - `异常入口快捷区`：跳转到业务看板、合同详情、收付款单列表、出入库列表

### 3.1.2 卡片设计
- `合同执行率`
  - 主值：`85.23%`
  - 副文案：`仅统计执行阶段合同`
  - 交互：点击跳转合同列表，默认筛选执行阶段合同
- `当日实收实付`
  - 双列展示：
    - `实收 ¥650,025.00`
    - `实付 ¥580,080.00`
  - 副文案：`按上海自然日统计`
- `库存周转`
  - 主值：`1.2865`
  - 副文案：`近30日出库 / 当前可用库存`
- `超阈值告警数`
  - 主值：`7`
  - 红色标签：`需处理`
  - 交互：点击直接进入 `ADM-BOARD-01`

### 3.1.3 图表与列表
- `异常分布`
  - 使用横向堆叠条形图
  - 类别固定：`待补录金额`、`校验失败`、`数量履约完成未关闭`
- `今日执行摘要`
  - 使用四宫格摘要卡
  - 字段：`今日实收`、`今日实付`、`今日入库量`、`今日出库量`

### 3.1.4 状态与文案
- 空态：`当前暂无可展示数据`
- 延迟态：`报表已超过 SLA，请稍后刷新或联系管理员`
- 版本提示：`当前口径版本：v1`

## 3.2 ADM-BOARD-01 业务看板

### 3.2.1 页面结构
1. 顶部栏
   - 页面标题：`业务看板`
   - 右侧信息：`更新时间`、`口径版本 v1`
2. 首屏三卡
   - `待补录金额`
   - `校验失败`
   - `数量履约完成未关闭`
3. 主体区域
   - Tab1：`资金待办`
   - Tab2：`库存阻塞`
   - Tab3：`合同待关闭`

### 3.2.2 列表结构
- 资金待办列表字段
  - 单据编号
  - 单据类型（收款单/付款单）
  - 关联合同编号
  - 当前状态
  - 创建时间
- 库存阻塞列表字段
  - 单据编号
  - 单据类型（入库单/出库单）
  - 关联合同编号
  - 当前状态
  - 创建时间
- 合同待关闭列表字段
  - 合同编号
  - 合同方向
  - 当前状态
  - 最近更新时间

### 3.2.3 交互要求
- 每行右侧提供 `查看详情` 按钮。
- 同一类型按“最早创建优先”排序，先暴露滞留问题。
- 顶部三卡点击后，直接切换到对应 Tab 并滚动到列表顶部。

### 3.2.4 异常表现
- `待补录金额`：黄色标签
- `校验失败`：红色标签
- `数量履约完成未关闭`：橙色标签
- 无数据空态：`当前暂无待处理异常`

## 4. 小程序原型说明

## 4.1 MINI-REPORT-01 轻量报表

### 4.1.1 页面结构
1. 顶部栏
   - 标题：`经营快报`
   - 右侧：`v1`
2. 第一屏四卡
   - `当日实收`
   - `当日实付`
   - `当日入库量`
   - `当日出库量`
3. 第二屏
   - `异常提醒` 单卡
   - 主值：`异常 3 项`
   - 副文案：`点击查看业务看板`

### 4.1.2 视觉细节
- 卡片为圆角矩形，主数字大字号，标签小字号。
- 用色：
  - 实收：深青色
  - 实付：赭红色
  - 入库量：墨绿色
  - 出库量：靛蓝色
  - 异常提醒：警示橙
- 底部保留空白，避免遮挡微信系统手势区。

### 4.1.3 权限边界
- 首版仅开放 `运营/财务/管理员（小程序）`。
- 客户、供应商、仓库不展示该页面入口。

### 4.1.4 文案与状态
- 正常态：`数据已更新`
- 延迟态：`数据更新中，请稍后再试`
- 空态：`今日暂无汇总数据`

## 5. 组件与文案细则
- 口径版本统一写法：`口径版本 v1`
- 更新时间统一写法：`更新时间：YYYY-MM-DD HH:mm`
- SLA 状态统一写法：
  - `正常`
  - `延迟`
- 异常标题统一：
  - `待补录金额`
  - `校验失败`
  - `数量履约完成未关闭`
