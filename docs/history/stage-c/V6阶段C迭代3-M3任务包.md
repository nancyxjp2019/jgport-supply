### 2.2 M3明确约束
- 销售订单创建允许两类入口：
  - 客户公司用户通过小程序创建；
  - 运营商公司 `运营/财务/管理员` 通过管理后台创建。
- 销售订单状态机本轮冻结为：`草稿 -> 待运营审批 -> 待财务审批 -> （驳回 或 已衍生采购订单）`。
- 运营审批驳回回退 `草稿`；财务审批驳回进入 `驳回`。
- 销售订单处于 `草稿` 或 `驳回` 时允许修改；`驳回` 态保存修改后必须自动回到 `草稿`。
- 销售订单财务审批通过时，必须绑定一个 `生效中` 的采购合同；系统自动生成的采购订单必须默认绑定该采购合同。
- 采购订单 `付款金额=0` 例外仅影响 `zero_pay_exception_flag` 与后续付款校验放行，不影响采购合同绑定。
- 财务审批通过后，本轮只生成采购订单实体，并为付款单、收款单写入待处理任务；不直接跨模块生成收付款单实体。
- 本轮不实现采购订单供应商确认、仓储执行、订单完成判定、附件上传和订单图谱。

## 3. 迭代3测试清单（先测后码）

| 用例ID | 用例名称 | 预期结果 |
|---|---|---|
| M3-T01 | 客户小程序创建销售订单草稿 | 返回 `草稿` 状态，单价必须等于销售合同单价 |

| M3-T08 | 财务审批通过且绑定采购合同 | 状态变为 `已衍生采购订单`，生成采购订单实体与 2 条收付款待处理任务 |

| M3-T10 | 财务审批驳回 | 状态变为 `驳回`，不生成采购订单与待处理任务 |
| M3-T11 | 采购订单详情查询 | 返回绑定的采购合同、来源销售订单、0付款例外标记 |
| M3-T12 | 采购订单0付款例外 | `zero_pay_exception_flag=true`，仍完成采购合同绑定与待处理任务写入 |
