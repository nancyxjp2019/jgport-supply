## 2. 迭代8第十八批范围（M8-18 库存与执行跟踪台首批）
- 目标：落地管理后台库存与执行跟踪台首批页面，补齐入库单/出库单追踪、校验失败回看与执行异常提示。
- 模块边界：本轮处理 `admin-web + backend库存单据最小查询接口补齐 + M5既有提交接口复用 + docs`；不新增库存余额台账、不新增批量提交、不新增合同关闭处置入口。
- 首批实现边界：
  - 管理后台新增 `/inventory` 页面与导航入口。
  - 后端补齐库存单据最小查询能力：入库单/出库单列表、详情只读回看。
  - 页面支持按单据类型、状态、来源筛选，展示合同/订单关联、油品、数量、仓库、来源回执信息。
  - 页面详情区输出执行异常提示：`校验失败`、`已终止`、`待提交` 等状态给出受控提示文案。
  - 演示模式提供可回归的库存执行 mock 数据，并新增单测。
- 本轮不实现：
  - 库存余额台账与库存结余报表
  - 退款核销、资金驳回、合同关闭处置入口
  - 批量提交、批量驳回、批量重试

## 2.2 M8-18 实施约束
- 页面文案统一中文，严格沿用 `M5` 既有状态口径：`草稿`、`待提交`、`校验失败`、`已过账`、`已终止`。
- 查询接口仅提供只读回看，不在本轮增加状态推进动作。
- 执行异常提示仅做受控提示，不替代服务端校验结论，不生成额外业务状态。
- 出库来源需区分 `SYSTEM` 与 `MANUAL`，并可回看 `source_ticket_no`、`manual_ref_no`。
