## 2. 迭代8第七批范围（M8-07 仓库执行回执）
- 目标：落地 `MINI-EXEC-01` 仓库执行回执页，支撑仓库角色在小程序中完成正常回执与手工补录出库。
- 模块边界：本轮只处理 `miniprogram/` 的仓库执行页、角色落点导航、表单校验、演示模式 mock 与 README；不改动后端业务规则，不进入订单发起、消息中心、供应商协同页面。
- 首批实现边界：
  - 新增 `pages/exec/index`。
  - 仓库角色登录后默认进入仓库执行回执页。
  - 支持“正常回执”和“手工补录”两种执行模式。
  - 复用现有 `/outbound-docs/warehouse-confirm`、`/outbound-docs/manual`、`/outbound-docs/{id}/submit` 接口形成移动端闭环。
  - 演示模式下提供本地模拟生效结果，便于页面评审。
- 本轮不实现：
  - 扫码带出订单/合同
  - 执行附件上传
  - 待执行列表查询
  - 供应商确认发货协同页

## 2.2 M8-07 实施约束
- 仅仓库角色默认进入执行回执页，其他角色保持原有默认工作页。
- 用户可见文案全部中文显示，不暴露英文状态和英文错误。
- 手工补录必须显式采集“销售合同ID + 销售订单ID + 油品ID + 手工回执号 + 原因”。
- 页面只做移动端高频动作，不加入复杂列表筛选和批量操作。
- 仓库执行页若被非仓库角色打开，必须给出中文阻断提示。
