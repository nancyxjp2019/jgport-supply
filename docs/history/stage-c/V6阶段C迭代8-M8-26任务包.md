## 2. 迭代8第二十六批范围（M8-26 每日闭环扫描与履约滞留扫描首批）
- 目标：补齐规则29在阶段 C 的首批落地，实现“每日闭环扫描 + 履约滞留扫描”的应用内日级补偿触发、审计留痕与看板/仪表盘告警闭环。
- 模块边界：本轮处理 `backend扫描服务 + 报表/看板告警补齐 + admin-web看板/仪表盘展示补齐 + docs`；不进入小程序通知中心、不建设独立调度平台、不做口径重算。
- 首批实现边界：
  - 后端新增两类扫描：`数量履约完成未关闭合同扫描`、`履约滞留合同扫描`。
  - 首批扫描采用“应用内日级补偿触发”实现：当管理后台仪表盘、业务看板或小程序轻量报表请求命中当日未扫描场景时，服务端自动补齐当日扫描；同一自然日重复请求按幂等键跳过。
  - 扫描命中结果必须写入业务审计日志，保留扫描日期、扫描类型、命中合同、判定依据与触发来源，形成可追溯留痕。
  - 业务看板、仪表盘、小程序轻量报表复用扫描结果展示异常计数；管理后台看板补齐“履约滞留”告警分类。
  - 履约滞留首批阈值冻结为：合同状态=`生效中` 且连续 `3` 个 `Asia/Shanghai` 自然日无新的 `contract_qty_effects` 累计记录。
- 本轮不实现：
  - 外部 cron / 独立 scheduler / 多实例分布式调度
  - 站内信、短信、小程序消息通知与告警升级编排
  - 扫描任务历史重跑中心、批量补跑、手工回填历史日历
  - 报表口径重算、历史版本回算与口径变更公告链路

## 2.2 M8-26 实施约束
- 用户可见文案、扫描结果说明、异常标题与审计备注统一中文显示，不向终端透传英文异常原文。
- 首批不新增独立“扫描任务中心”页面；异常展示统一复用仪表盘、业务看板与轻量报表既有入口。
- 闭环扫描只覆盖 `数量履约完成` 且未进入 `已关闭/手工关闭/已归档` 的合同；不得把已关闭合同重复记为待处理异常。
- 履约滞留扫描只覆盖 `生效中` 合同；首批阈值固定为 `3` 个上海自然日，不在本轮扩展参数中心配置。
- 同一扫描类型、同一合同、同一上海自然日只允许生成一条有效审计留痕，避免重复告警与重复审计。
