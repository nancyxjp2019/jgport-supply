## 2. 迭代8第十一批范围（M8-11 消息回看与深链一致性）
- 目标：补强 `MINI-MSG-01` 与已开放业务页之间的深链一致性，确保消息点击后能直接定位到对应业务页和处理上下文。
- 模块边界：本轮只处理 `miniprogram/` 的统一深链工具、消息来源页回看、订单页/快报页/仓库执行页入口参数解析与测试文档；不新增后端接口，不新增消息表，不进入供应商真实业务页。
- 首批实现边界：
  - 统一 `消息中心 -> 订单页/快报页/执行页` 的深链拼装规则。
  - 订单页支持按消息来源自动定位到查询态、状态筛选或编辑态。
  - 经营快报支持按消息来源自动展开异常明细，并定位当前关注异常类型。
  - 仓库执行页支持按消息来源自动定位“正常回执 / 手工补录”模式。
  - 业务页展示“来源回看”提示，保证用户知道当前由消息中心进入。
- 本轮不实现：
  - 服务端消息持久化与已读回写
  - 供应商真实业务消息流
  - 供应商确认发货真实业务页
  - 后台消息策略配置

## 2.2 M8-11 实施约束
- 本轮只补消息链路一致性，不新增任何业务规则与后端存储。
- 用户可见文案全部中文显示，来源提示、异常关注和页面定位说明不得直出英文。
- 深链规则必须统一收口到公共工具，不允许继续在消息聚合函数中散落硬编码 URL。
