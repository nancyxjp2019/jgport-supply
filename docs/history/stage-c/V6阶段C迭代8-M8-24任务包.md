## 2. 迭代8第二十四批范围（M8-24 导出编排任务中心首批）
- 目标：承接 `M8-21` 的页面内同步导出，补齐“异步导出任务 + 导出历史 + 下载/失败重试”的后台任务中心闭环。
- 模块边界：本轮处理 `backend报表导出任务编排 + admin-web任务中心页 + 本地导出文件目录 + docs`；不进入小程序、不建设跨报表通用编排平台、不引入消息通知中心。
- 首批实现边界：
  - 后端新增多维报表导出任务接口：创建任务、任务列表、结果下载、失败重试。
  - 导出任务首批仅支持 `多维报表当前筛选快照 + CSV`，继续沿用 `metric_version=v1`。
  - 任务状态冻结为：`待处理 -> 处理中 -> 已完成 / 已失败`，并保留失败原因、重试次数、下载次数与完成时间。
  - 导出文件首批落盘到 V6 本地目录，由任务中心统一回看与下载，不再以页面内即时下载作为唯一入口。
  - 管理后台新增“导出任务中心”页面与导航入口，多维报表页改为“创建导出任务 + 跳转任务中心”。
  - 演示模式补齐任务中心 mock 与单测，后端补齐任务接口与回归测试。
- 本轮不实现：
  - 跨报表批量导出编排与优先级队列
  - 消息通知、站内信、短信或小程序导出提醒
  - 通用离线文件中心、自动清理调度、文件过期策略中心
  - 分布式队列、跨实例任务协调与大文件分片导出

## 2.2 M8-24 实施约束
- 页面、提示、状态与失败原因统一中文显示，不得向终端透传英文异常原文。
- 导出任务权限沿用 `M8-23` 后端边界：仅 `finance/admin + operator_company + admin_web` 可创建、下载、重试；`operations` 仅保留报表查询。
- 导出任务必须固化创建时的筛选快照，任务执行与重试均按快照重放，不允许前端在下载时重新拼参。
- 首批仅支持多维报表单任务 CSV 导出，不扩展到仪表盘、看板、合同、资金等其他报表来源。
- 首批采用应用内后台任务 + 本地文件目录实现异步编排，不补分布式 worker 与多实例协调。
