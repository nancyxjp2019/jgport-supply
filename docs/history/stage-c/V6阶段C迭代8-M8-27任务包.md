## 2. 迭代8第二十七批范围（M8-27 汇总报表口径重算首批）
- 目标：补齐规则58在阶段 C 的首批落地，实现汇总报表当前快照的手工重算任务创建、后台执行、历史回看与审计留痕闭环。
- 模块边界：本轮处理 `backend重算任务服务 + 汇总快照强制重建 + admin-web任务入口/历史页补齐 + docs`；不进入历史日期重算、不进入多维报表历史回算、不建设口径公告中心。
- 首批实现边界：
  - 后端新增汇总报表重算任务：仅支持 `dashboard_summary`、`board_tasks`、`light_overview` 三类汇总报表。
  - 重算任务采用“管理后台手工触发 + 应用内后台执行”方式落地；执行成功后必须强制新增一条 `report_snapshots` 快照，而不是仅复用原快照。
  - 任务记录必须保留：目标报表集合、`metric_version`、发起原因、发起人身份、任务状态、失败原因、完成时间与结果快照时间。
  - 管理后台补齐“汇总报表重算任务中心”或与现有报表任务中心并列入口，支持任务创建、历史回看、失败重试。
  - 首批继续沿用 `metric_version=v1`，不在本轮扩展新版本公式、历史日期回算与公告发布。
- 本轮不实现：
  - 历史自然日区间批量回算
  - 多维报表筛选快照重算与导出文件重放
  - 新增 `v2`/`v3` 等口径公式及公告发布中心
  - 跨实例分布式队列、通知中心与自动定时重算

## 2.2 M8-27 实施约束
- 所有用户可见文案、错误提示与任务状态统一使用中文，不向前端透传英文内部异常原文；任务原因必须填写审计可读说明。
- 首批仅开放 `finance/admin + operator_company + admin_web` 创建或重试重算任务；`operations` 仅保留只读查看能力。
- 同一操作人、同一报表集合、同一 `metric_version`、同一原因命中未终态任务时，必须直接复用原任务，不得重复排队。
- 重算执行成功后必须生成新的 `report_snapshots` 记录，并回填到任务结果，避免仅更新旧快照时间导致审计链丢失。
- 首批仅重算当前时点汇总快照，不在本轮引入历史日期参数、历史版本回算与多维筛选快照补算。
