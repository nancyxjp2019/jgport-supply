## 2. 迭代5范围（M5库存执行模块）
- 目标：落地入库单、出库单、仓库正常回执与手工补录出库的实体链路，并完成合同履约数量累计与防重。
- 模块边界：`M5` 只处理出入库单据、履约累计与数量履约完成状态；不进入金额闭环、自动关闭、库存余额报表。
- 首版实现边界：本轮不建设独立库存台账/库存余额表，先完成“单据 -> 履约累计 -> 过账状态留痕”最小闭环。

## 2.2 M5实施约束
- 采购合同若包含多个油品明细，系统按合同油品明细逐条生成入库单草稿。
- 入库单、出库单必须显式记录 `oil_product_id` 与 `warehouse_id`；不得仅靠上游单据隐式推导。
- 仓库正常流程出库单必须记录 `source_ticket_no` 并生成唯一 `idempotency_key`。
- 手工补录出库单必须同时绑定 `销售合同 + 销售订单 + 油品`，且 `manual_ref_no` 在合同+油品维度唯一。
- 所有出库单仅允许基于状态为 `已衍生采购订单` 或 `执行中` 的销售订单生成，不得绕过订单审批主链。
- 入库/出库提交时，若合同已进入 `数量履约完成`，单据转 `已终止` 并阻断继续生效。
- 入库/出库提交若超出合同超量履约阈值，单据转 `校验失败`；允许调整数量后再次提交。
- 同一出入库单只允许写入一次 `contract_qty_effects`；命中重复键时按幂等跳过并写审计。
- 本轮不实现：
  - 独立库存余额表、库存结余报表
  - 采购订单/销售订单完成态联动
  - 供应商确认发货状态机
  - 合同金额闭环与自动关闭
