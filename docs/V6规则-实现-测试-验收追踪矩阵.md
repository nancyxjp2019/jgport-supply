# V6 规则-实现-测试-验收追踪矩阵

## 1. 文档定位
- 文档状态：`生效`
- 目标：把本轮面向“生产可用版本”的关键规则、实现对象、测试覆盖、验收证据与遗留去向统一挂接，避免开发遗漏、测试遗漏、验收遗漏与口头降级。
- 使用边界：本文不替代 `docs/需求方案.md` 的规则正文；本文负责追踪关键规则在实现、测试、验收与上线阶段的落点。

## 2. 使用规则
- 每个关键规则至少维护一条追踪记录。
- 未完成“规则 -> 设计 -> 实现 -> 测试 -> 验收”映射的事项，不得宣称已完成。
- 若当前仅冻结口径尚未编码，`当前状态` 必须明确写 `已冻结，未实现`。
- 任何降级、遗留、延后事项，必须写入 `遗留去向`，不得口头承接。
