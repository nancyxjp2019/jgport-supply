## 2. 迭代8第二十二批范围（M8-22 按钮级权限首批）
- 目标：落地管理后台按钮级权限首批能力，补齐运营/财务/管理员在页面动作层的最小权限收口。
- 模块边界：本轮处理 `admin-web按钮权限收口 + docs`；不改后端角色鉴权链路、不新增组织级权限模型、不新增按钮审计中心。
- 首批实现边界：
  - 新增前端按钮权限工具，统一管理首批动作权限编码。
  - 订单处理台按角色收口审批按钮：运营仅可执行运营审批，财务仅可执行财务审批，管理员可执行两类审批。
  - 资金处理台、退款核销台按钮收口为财务/管理员可执行，运营只读回看。
  - 多维报表导出按钮收口为财务/管理员可执行，运营只读回看。
  - 页面在无权限时展示只读提示，不展示“可执行但会失败”的误导状态。
- 本轮不实现：
  - 后端细粒度按钮鉴权接口
  - 组织、岗位、数据域三维权限模型
  - 权限配置后台与权限变更审批流
