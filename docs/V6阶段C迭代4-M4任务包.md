# V6 阶段C迭代4（M4）任务包（实施中）

## 1. 当前阶段判定
- 当前阶段：`阶段C迭代4（M4）实施中`。
- 进入条件：`M3` 已完成并通过模块门禁，销售订单财务审批后已能生成采购订单与收付款待处理任务。

## 2. 迭代4范围（M4资金单据模块）
- 目标：落地收款单、付款单实体与手工补录入口，消费 `M2/M3` 产生的待处理任务，形成真实资金单据与基础追溯链路。
- 当前批次边界：本轮只做“实体生成 + 手工补录 + 上下游关系”；不在本轮实现核销、退款、规则14阈值放行引擎和附件上传落库。

### 2.1 M4子模块拆分

| 子模块ID | 子模块名称 | 输入 | 输出 | 依赖 | 状态流转 | 失败回滚 |
|---|---|---|---|---|---|---|
| M4-01 | 合同保证金单据实体化 | `contract_effective_tasks` | 收款单/付款单草稿 | M2 | 待处理 -> 已生成 | 同事务回滚 |
| M4-02 | 订单实收实付单据实体化 | `sales_order_derivative_tasks` | 收款单/付款单草稿 | M3 | 待处理 -> 已生成 | 同事务回滚 |
| M4-03 | 手工补录付款单 | 合同、采购订单、金额 | 付款单草稿 | M2、M3 | 草稿 | 同事务回滚 |
| M4-04 | 手工补录收款单 | 合同、销售订单、金额 | 收款单草稿 | M2、M3 | 草稿 | 同事务回滚 |
| M4-05 | 资金单据追溯关系 | 合同/订单/采购订单/资金单据 | `doc_relations` 关系链 | M2、M3、M4-01~M4-04 | 只写关系 | 同事务回滚 |

### 2.2 M4当前批次明确约束
- 自动生成的合同保证金收付款单在本轮先落为 `草稿`，金额初始化为 `0.00`，由后续财务确认流程补齐。
- 销售订单衍生的收付款单在本轮先根据任务金额生成 `草稿` 单据，不在本轮直接做确认与核销。
- 销售订单衍生采购订单 `付款金额=0` 场景，付款单草稿必须写入例外标识与中文提示文案，保留后续补录能力。
- 手工补录付款单必须同时绑定 `采购合同 + 采购订单`，且两者必须互相匹配。
- 手工补录收款单必须同时绑定 `销售合同 + 销售订单`，且两者必须互相匹配。
- 本轮先实现 `doc_relations` 的基础追溯，至少覆盖：
  - 合同 -> 保证金收付款单
  - 销售订单 -> 收款单
  - 采购订单 -> 付款单
  - 销售订单 -> 采购订单
- 本轮不实现：
  - `/receipt-docs/{id}/confirm`
  - `/payment-docs/{id}/confirm`
  - 凭证附件落库
  - 退款/核销
  - 规则14阈值放行自动判定
