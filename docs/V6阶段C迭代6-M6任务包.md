## 2. 迭代6范围（M6履约闭环与关闭模块）
- 目标：落地合同金额闭环计算、自动关闭、手工关闭核心收口动作。
- 模块边界：`M6` 只处理合同关闭判定、关闭收口、关闭差异记录与关闭后阻断；不进入日报扫描、看板告警、报表展示。
- 首批实现边界：本轮不建设每日闭环扫描任务、不推进 `已归档` 归档态、不处理订单完成态联动。

## 2.2 M6实施约束
- 自动关闭仅在合同已达到 `数量履约完成` 且金额闭环满足 `|差额| <= 0.01` 时触发。
- 金额闭环必须按合同方向分开计算：销售只看收款净额，采购只看付款净额。
- 自动关闭与手工关闭均需执行关闭收口：未终态的收付款单、入库单、出库单统一转 `已终止` 并写审计。
- 手工关闭仅允许在合同达到 `数量履约完成` 后执行，不允许跳过数量履约直接关闭合同。
- 手工关闭必须同时满足：原因必填、确认口令=`MANUAL_CLOSE`、审计留痕。
- 手工关闭差异记录至少包含：关闭差异金额、按油品的关闭差异数量明细。
- 合同进入 `已关闭/手工关闭` 后，服务端必须阻断新的资金补录、出库补录、出入库再次提交与0金额再次确认。
- 本轮不实现：
  - 每日闭环扫描与履约滞留扫描任务
  - 看板告警、仪表盘闭环告警
  - 合同 `已归档` 归档态
  - 销售订单/采购订单完成态联动
