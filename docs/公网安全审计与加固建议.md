# 隽港顺达供应链项目 公网安全审计与加固建议

## 1. 文档信息
- 版本：V5
- 更新日期：2026-03-01
- 适用范围：V5 阶段后台管理页面（`/admin-console`）、后端 API（`/api/v1`）、部署文档与示例配置

## 2. 已完成加固项
1. 超级管理员登录二次验证增加限流（TOTP/恢复码），连续高频请求会返回 `429`。
2. 后端统一增加基础安全响应头（`X-Frame-Options`、`X-Content-Type-Options` 等）。
3. 后台令牌存储由 `localStorage` 调整为 `sessionStorage`，降低 XSS 持久化窃取风险。
4. 附件下载接口已改为必须登录访问，客户角色增加订单范围校验，阻断匿名读取与跨客户越权。
5. 示例配置与部署文档中的真实密钥已替换为占位符，避免再次泄露。

## 3. 当前高优先级风险（建议尽快处理）
1. Nginx 层尚未完整启用限流
- 当前应用层已对高风险接口增加限流，但公网入口仍建议叠加 Nginx `limit_req`。
- 建议：按部署手册示例开启并压测阈值。

## 4. 中优先级风险
1. 超级管理员 Challenge Token 可在有效期内重复使用
- 风险：若令牌泄露，在过期前可能被重放。
- 建议：引入一次性 nonce（验证成功即失效），并记录挑战状态。

2. `file_storage_backend=local` 时存在静态附件路径
- 风险：历史 `local` 附件若使用 `/media/...` 直链，仍可能被直接访问。
- 建议：生产环境优先使用 `oss` 私有桶并保持 `oss_base_url` 为空，统一走鉴权代理地址。

3. API 文档对公网默认可见
- 风险：暴露接口结构，增加被扫描和定向攻击概率。
- 建议：生产环境关闭 `/docs`、`/redoc`，仅在内网或运维白名单开放。

## 5. 部署与运维建议
1. 强制 HTTPS，仅保留 443 对外服务，80 全量跳转到 HTTPS。
2. 云防火墙/安全组仅放行 80、443（或仅 443），数据库与 SSH 限制来源 IP。
3. 所有密钥启用定期轮换策略（建议 90 天），并保留轮换记录。
4. 日志接入告警：`401/403/429` 异常突增、登录失败峰值、上传失败峰值。
5. 定期执行依赖漏洞扫描（Python 包、Nginx、系统包）。

## 6. 建议迭代顺序
1. 第一优先：文件下载鉴权 + Nginx 限流。
2. 第二优先：Challenge Token 一次性机制 + 生产关闭 API 文档。
3. 第三优先：全链路安全基线（WAF、漏洞扫描、密钥轮换自动化）。
