# V6 文档导航

## 1. 当前生效基线
- `docs/需求方案.md`：业务规则唯一基线，所有规则必须标注 `生效`、`弃用`、`规划中`。
- `docs/V6项目总控台账.md`：项目经理/负责人视角的宏观总控、阶段门禁与唯一焦点基线。
- `docs/V6生产可用版本治理总纲.md`：从“当前可联调版本”推进到“生产可用版本”的唯一宏观治理总纲。
- `docs/V6阶段E-验收发布与上线清单.md`：阶段 E 验收、发布、回滚与观察期执行基线。
- `docs/V6规则-实现-测试-验收追踪矩阵.md`：生产化开发阶段的规则追踪、实现映射、测试与验收基线。
- `docs/新项目重建基座说明-Vue3-Vben-Postgres.md`：V6 技术基座、目录与部署说明。
- `docs/V6阶段A-流程图状态机与UI原型清单.md`：阶段 A 流程、状态机与原型冻结清单。
- `docs/V6阶段B-详细设计与原型冻结包.md`：阶段 B 详细设计冻结基线。
- `docs/V6阶段D-联调与综合回归清单.md`：旧版“可联调”范围的联调基线，当前作为后续按生产范围重启阶段 D 的输入文档。
- `docs/V6现行高保真原型说明.md`：当前仍被 `M8` 系列实现与任务包引用的高保真原型补充说明。

## 2. 历史归档目录
- `docs/history/pre-stage-c/`：阶段 C 启动前的统筹、检查、追踪矩阵与题库材料。
- `docs/history/stage-c/`：阶段 C 各迭代任务包与已完成交付记录。
- `docs/history/stage-d/`：旧版阶段 D 联调执行记录、手工联调清单与回归留痕材料。

## 3. 规划目录
- `docs/planning/`：阶段性缺口盘点、路线图与后续批次建议；当前主入口为 `docs/planning/V6阶段C后续能力路线图.md`。

## 4. 维护约定
- `docs/` 根目录仅保留当前仍需持续更新的基线文档。
- 面向“生产可用版本”的宏观判断，统一以 `docs/V6生产可用版本治理总纲.md`、`docs/V6阶段E-验收发布与上线清单.md`、`docs/V6规则-实现-测试-验收追踪矩阵.md` 为准。
- 已完成任务包、一次性检查报告、历史交付记录统一归档到 `docs/history/`。
- 阶段性路线与缺口分析统一放到 `docs/planning/`，避免和基线文档混放。
