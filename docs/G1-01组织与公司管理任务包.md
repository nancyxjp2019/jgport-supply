# G1-01 组织与公司管理任务包

## 1. 文档定位
- 文档状态：`生效`
- 目标：作为 `G1-01 组织与公司管理` 的当前模块任务包，冻结本子模块的输入、输出、依赖、状态流转、失败回滚与首批实现范围。
- 使用边界：本文仅约束 `G1-01` 首批实现，不替代 `docs/需求方案.md`、`docs/V6项目总控台账.md`、`docs/V6生产可用版本治理总纲.md`、`docs/V6规则-实现-测试-验收追踪矩阵.md`。

## 2. 对应规则
- 规则 `59`：管理后台真实业务测试前置治理能力。
- 规则 `60`：管理后台组织与公司管理首批补齐范围。
- 设计基线：`ADM-ORG-01`。

## 3. 上下游边界

### 3.1 输入
- 管理后台管理员身份：仅 `admin + operator_company + admin_web` 可访问。
- 公司主档字段：公司编码、公司名称、公司类型、归属运营商、备注。
- 状态动作：启用、停用。

### 3.2 输出
- 后端公司主档模型与接口。
- 管理后台 `ADM-ORG-01` 页面，支持列表、详情、创建、编辑、启停用。
- 审计留痕：公司创建、编辑、启停用写入业务审计日志。

### 3.3 依赖
- 现有权限校验：`require_actor`。
- 现有审计模型：`business_audit_logs`。
- 当前开发环境认证仍沿用 `demo/proxy` 开发模式；本模块不同时收口正式登录。

### 3.4 状态流转
- 公司状态首批仅支持：`启用 -> 停用`、`停用 -> 启用`。
- 运营商公司首批不绑定上级公司。
- 客户/供应商/仓库公司首批必须绑定一个启用中的运营商公司作为归属关系。

### 3.5 失败回滚
- 创建/编辑/启停用失败时，当前请求整体回滚，不允许出现“公司状态已改但审计未写”或“审计已写但公司未更新”的半成功状态。
- 停用动作若命中“仍存在启用中的下级公司”，必须整单阻断并返回明确中文提示。

## 4. 首批实现范围
- 后端：新增公司主档表、迁移、模型、Schema、Service、Router、自动化测试。
- 前端：新增 `ADM-ORG-01` 页面、路由、导航入口、演示数据、页面自动化测试。
- 文档：同步更新 `docs/README.md`、`docs/V6项目总控台账.md`、`docs/V6规则-实现-测试-验收追踪矩阵.md`。

## 5. 首批非目标
- 不在本模块补齐用户与角色绑定、主数据中心、审批模板与参数配置。
- 不做复杂组织树、多层法人穿透、批量导入导出、物理删除。
- 不在本模块直接回填合同、订单、资金、库存等历史业务单据的外键关联。

## 6. 验收与测试口径
- 后端：覆盖组织建档、归属关系校验、启停用阻断、审计留痕。
- 前端：覆盖列表加载、创建、编辑、启停用、角色边界。
- 构建门禁：`backend` 自动化通过，`admin-web` 测试与构建通过。
