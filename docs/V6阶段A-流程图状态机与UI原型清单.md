# V6 阶段A交付件：流程图、状态机与UI原型清单（已冻结）

## 1. 文档定位
- 文档状态：`已冻结`
- 目标：冻结业务流、状态流、端能力边界，为阶段B详细设计与原型冻结提供输入。
- 适用范围：管理后台（全业务）与小程序（移动轻量业务）。
- 规则基线：`docs/需求方案.md` 当前 `1~69` 条及“业务目标与场景/角色与权限边界”章节。

## 1.1 角色口径说明
- 业务角色固定为：客户、运营、财务、供应商、仓库；系统管理员归属运营商公司。
- 本文中“管理层/审计”仅表示使用视角，不新增独立业务角色，实际由运营/财务/管理员权限组合承载。
- 客户、供应商、仓库角色用户仅在小程序端处理业务，不开放管理后台登录权限。

## 2. 端到端流程图

### 2.1 采购合同主流程（合同生效触发）
```mermaid
flowchart TD
    A[财务录入采购合同] --> B[审批通过]
    B --> C[合同生效]
    C --> D[系统自动生成付款单-保证金（写入上下游ID）]
    C --> E[系统按油品明细自动生成入库单-草稿（写入上下游ID）]
    E --> F[仓库录入实际入库数量]
    F --> F0{合同是否已数量履约完成}
    F0 -->|是| F2[阻断入库单生效并告警]
    F0 -->|否| F1{累计实际入库量是否超过合同上限阈值}
    F1 -->|是| F2
    F1 -->|否| G[入库单提交生效]
    G --> H[计入库存与合同履约数量]
    D --> I[财务确认实付]
    I --> J[计入采购合同资金台账]
```

### 2.2 销售合同与销售订单主流程
```mermaid
flowchart TD
    A[财务录入销售合同] --> B[审批通过]
    B --> C[销售合同生效]
    C --> D[系统自动生成收款单-保证金（写入上下游ID）]
    E[客户录入销售订单] --> F[运营审批]
    F --> G[财务审批通过]
    G --> H[系统自动生成采购订单（写入上下游ID）]
    G --> I[系统自动生成付款单-采购订单实付金额（写入上下游ID）]
    G --> J[系统自动生成收款单-销售订单实收金额（写入上下游ID）]
    H --> K[供应商上传盖章发货指令单并确认]
    K --> K0[采购订单进入供应商已确认（含付款确认/付款校验，付款金额可为0）]
    K0 --> L1[仓库正常流程实际出库]
    L1 --> M1[系统自动生成出库单-待生效（写入上下游ID）]
    K0 --> N1[仓库手工补录出库申请]
    N1 --> M2[手工补录出库单-待生效（必须绑定销售合同+销售订单）]

    M2 --> U0
    U0 -->|是| V[阻断出库单生效并告警]
    U0 -->|否| U{累计实际出库量是否超过合同上限阈值}
    U -->|是| V
    U -->|否| W[出库单生效并过账]
    W --> W1[写入履约累计防重流水（幂等）]
    W1 --> N[合并计入销售合同履约数量]
    J --> R{销售订单收款金额是否为0}
    R -->|是| S[按零金额例外流转并记录免凭证原因]
    R -->|否| T[按正常收款流转并校验凭证]
    D --> O[计入销售合同资金台账]
    I --> P[计入采购合同资金台账]
    S --> Q[计入销售合同资金台账]
    T --> Q[计入销售合同资金台账]
```

### 2.3 销售订单衍生采购订单“零付款”例外流程
```mermaid
flowchart TD
    A[销售订单财务审批通过] --> B[自动生成采购订单]
    B --> C{付款金额是否为0}
    C -->|否| D[按正常付款单流程提交]
    C -->|是| E[无条件放行（暂定，不校验保证金放行阈值）并允许继续提交采购订单]

    F --> G[后续财务手工补录付款单]
    G --> H[补录付款单必须绑定采购合同+采购订单]
    H --> I[更新采购合同资金台账]
```

### 2.4 销售订单自动收款单（零金额/正常）流程
```mermaid
flowchart TD
    A[销售订单财务审批通过] --> B[系统自动生成收款单]
    B --> C{收款金额是否为0}
    C -->|是| D{是否满足保证金放行阈值}
    D -->|是| E[允许零金额提交]
    E --> F[免上传凭证]
    F --> G[记录免凭证原因/操作人/时间]
    G --> H[收款单进入已确认并纳入资金台账]
    D -->|否| I[禁止零金额提交并提示补齐金额]
    I --> L[收款单转待补录金额并阻断提交]
    L --> J[补录为非0金额后按正常收款流程提交]
    C -->|否| J[按正常收款流程提交]
    J --> K[上传凭证并通过审核]
    K --> H
```

### 2.5 合同关闭与手工关闭流程
```mermaid
flowchart TD
    A[持续履约中] --> B{数量履约是否完成}
    B -->|否| A
    B -->|是| C{合同方向}
    C -->|销售| D[校验销售闭环: Σ实际出库量×合同单价 vs 累计收款净额]
    C -->|采购| E[校验采购闭环: Σ实际入库量×合同单价 vs 累计付款净额]
    D --> D1{是否满足}
    E --> E1{是否满足}
    D1 -->|是| F[自动进入合同最终关闭]
    E1 -->|是| F
    F --> F1[终止剩余未终态草稿/待处理单据并审计]
    D1 -->|否| G[仅授权用户可手工关闭]
    E1 -->|否| G
    G --> H[填写原因+二次确认]
    H --> I[锁定合同与关联单据]
    I --> J[终止未生效单据并记录原因]
    J --> K[释放未执行库存预占/预留]
    K --> L[生成手工关闭差异记录]
    L --> M[合同及关联单据标记手工关闭]
```

### 2.6 每日扫描与告警流程（闭环/履约）
```mermaid
flowchart TD
    A[每日定时任务触发] --> B[扫描数量履约完成且未关闭合同]
    A --> C[扫描生效中且履约长时间未变化合同]
    B --> D[生成闭环待处理告警]

    E --> F
    F --> G[写入任务审计日志]
    G --> H{是否重复触发}
    H -->|是| I[幂等跳过]
    H -->|否| J[完成本次扫描]
```

### 2.7 单据生成与上下游ID写入流程（强制）
```mermaid

    B --> C[写入source_doc_id与业务方向]
    C --> D[写入upstream_doc_ids/downstream_doc_ids]
```

## 3. 核心状态机

### 3.1 合同状态机（采购/销售通用骨架）
```mermaid
stateDiagram-v2
    [*] --> 草稿
    草稿 --> 待审批: 提交审批
    待审批 --> 草稿: 驳回
    待审批 --> 生效中: 审批通过
    生效中 --> 数量履约完成: 累计履约数量达到合同数量
    生效中 --> 手工关闭: 授权关闭
    数量履约完成 --> 待最终关闭校验: 触发金额闭环校验
    待最终关闭校验 --> 销售闭环校验中: 合同方向=销售
    待最终关闭校验 --> 采购闭环校验中: 合同方向=采购
    销售闭环校验中 --> 已关闭: 销售闭环满足
    采购闭环校验中 --> 已关闭: 采购闭环满足
    销售闭环校验中 --> 手工关闭: 销售闭环不满足且授权关闭
    采购闭环校验中 --> 手工关闭: 采购闭环不满足且授权关闭
    手工关闭 --> 已归档: 审计归档完成
    已关闭 --> 已归档
```

### 3.2 销售订单状态机
```mermaid
stateDiagram-v2
    [*] --> 草稿
    草稿 --> 待运营审批: 提交
    待运营审批 --> 草稿: 驳回
    待运营审批 --> 待财务审批: 运营通过
    待财务审批 --> 驳回: 财务驳回
    待财务审批 --> 已衍生采购订单: 财务通过并自动生成采购订单/收付款单
    已衍生采购订单 --> 执行中: 仓储与发运执行
    执行中 --> 已完成: 实际出库完成
    驳回 --> 草稿: 修改后重提
```

### 3.3 采购订单状态机（含零付款例外）
```mermaid
stateDiagram-v2
    [*] --> 已创建
    已创建 --> 待供应商确认: 下发
    待供应商确认 --> 供应商已确认: 上传盖章发货指令单并确认，完成付款确认/付款校验
    供应商已确认 --> 执行中: 发运/仓储执行
    执行中 --> 已完成: 对应出入库完成
```

### 3.4 收付款单状态机
```mermaid
stateDiagram-v2
    [*] --> 草稿
    草稿 --> 待补录金额: 零金额校验不通过（规则14场景）
    草稿 --> 已确认: 确认并通过校验
    待补录金额 --> 已确认: 补录金额/凭证后再次确认
    已确认 --> 已核销: 与合同/订单核销完成
    草稿 --> 已终止: 关联合同手工关闭
    待补录金额 --> 已终止: 关联合同手工关闭
```

### 3.5 出入库单状态机
```mermaid
stateDiagram-v2
    [*] --> 草稿
    草稿 --> 待提交: 录入实际数量
    待提交 --> 已终止: 合同已数量履约完成
    待提交 --> 校验失败: 超量履约阈值校验失败
    校验失败 --> 待提交: 调整数量后重提
    待提交 --> 已生效: 提交并校验通过
    已生效 --> 已过账: 库存流水过账
    草稿 --> 已终止: 关联合同手工关闭
    待提交 --> 已终止: 关联合同手工关闭
```

## 5. 管理后台与小程序能力矩阵（冻结建议）

| 能力域 | 管理后台 | 小程序 | 决策说明 |
|---|---|---|---|
| 合同全生命周期 | 支持 | 仅查询 | 合同建立与关闭属于高风险操作 |
| 销售订单 | 支持全流程 | 支持发起/查询/轻审批 | 移动端保留高频动作 |
| 采购订单 | 支持全流程 | 查询进度 + 首批附件回传 + 单笔发货确认 | 小程序仅开放低风险协同动作，资金确认与异常处置仍保留后台 |
| 收付款单 | 支持全流程 | 仅查看结果 | 财务核销与凭证审核必须在后台 |
| 出入库单 | 支持全流程 | 支持执行回执与确认 | 执行动作可移动化，规则配置不可移动化 |
| 阈值与字典配置 | 支持 | 不支持 | 避免误操作造成全局风险 |
| 手工关闭 | 支持（授权） | 不支持 | 高风险动作需强审计场景 |
| 仪表盘与看板 | 支持 | 轻量摘要 | 管理决策在后台完成 |
| 多维报表 | 支持 | 不支持 | 多维分析与钻取对交互复杂度要求高 |

- 端登录边界冻结：客户、供应商、仓库角色仅允许小程序登录，不开放管理后台登录。

## 6. 原型产出清单（阶段A）
- 后台高保真原型：仪表盘、业务看板、合同详情链路页、收付款单页、出入库执行页。
- 小程序中保真原型：待办页、订单进度页、供应商采购进度页（含附件回传首批态）、执行回执页、轻量报表页。
- 原型状态覆盖：正常态、空态、异常态、审批态、超阈值告警态。

## 7. 评审通过标准（阶段A出口）
- 业务方确认：流程图覆盖采购、销售、资金、库存、关闭五大链路。
- 财务确认：收付款单据化、零付款例外、金额闭环口径一致。
- 财务确认：销售订单自动收款单“0金额/非0金额”两条分支流程完整且口径一致。
- 财务确认：采购/销售金额闭环分口径校验路径清晰，不存在混算。
- 仓储确认：入库生效前后口径一致，出库来源双通道（系统+手工）一致。
- 仓储确认：入库/出库生效前均执行合同超量履约阈值校验，超限阻断可见。
- 仓储确认：合同进入数量履约完成后，新增出入库生效被阻断。
- 财务确认：保证金在闭环净额中不重复计入，退款口径可追溯。
- 审计确认：双通道履约累计具备幂等防重，不出现重复累计。
- 管理确认：后台/小程序边界清晰，仪表盘与看板指标范围明确。
- 权限确认：角色与公司归属口径一致（客户/运营/财务/供应商/仓库+管理员），不存在跨公司越权。

## 8. 冻结结果（阶段A）
- 本文新增的后台治理页面与状态口径属于阶段A冻结输入，用于约束后续设计与开发边界，不代表当前代码仓库已经具备对应页面实现。
