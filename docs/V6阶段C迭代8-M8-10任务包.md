## 2. 迭代8第十批范围（M8-10 消息中心首批）
- 目标：落地 `MINI-MSG-01`，统一承接当前已开放业务链的首批消息提醒、异常提示和状态通知。
- 模块边界：本轮只处理 `miniprogram/` 的消息中心页、消息聚合纯函数、待办入口跳转、设备侧已读状态与文档；不新增后端消息表，不新增服务端消息接口，不进入消息策略配置。
- 首批实现边界：
  - 新增 `pages/msg/index`。
  - 客户消息基于真实订单状态聚合。
  - 运营/财务/管理员消息基于真实轻量报表异常摘要聚合。
  - 仓库、供应商、未知角色消息按当前已开放边界输出中文提示。
  - 支持 `全部 / 未读 / 已读` 切换与本地设备侧已读状态保存。
  - 待办页增加消息中心入口。

  - 消息订阅、短信、微信订阅通知
  - 供应商真实业务消息流

## 2.2 M8-10 实施约束
- 首批消息中心只做“聚合 + 查看 + 本地已读”，不进入服务端消息持久化。
- 用户可见文案必须全中文，且消息标题、摘要、空态、错误态均不得直出英文。
- 消息中心不暴露新的业务口径，不新增业务规则；消息内容必须基于当前已生效的订单状态、报表异常计数和角色边界。
- 供应商角色如无真实业务消息，不得展示伪造数量，只允许展示中文边界提示。
- 仓库消息需保留执行入口跳转，不能只展示静态说明。
